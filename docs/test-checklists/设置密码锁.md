# 功能名：游戏端设置页访问密码锁

> 背景：游戏端「设置」页有整库导入、清理照片等危险操作，孩子容易误入乱点。
> 加一道密码锁：设过密码后进设置页要先输密码；可用旧密码修改；忘记密码用「家长算术验证」重设。
>
> ⚠️ 定位：只挡小孩，不是高强度安全。密码存 localStorage 的轻量哈希，懂开发者工具的人能绕过——但孩子不会。
> ⚠️ localStorage 按「App 上下文」隔离：请在**孩子实际使用的那个入口**（iPad 主屏 PWA 图标）里设置密码，
>     别在 Safari 浏览器里设、指望 PWA 生效。

## 桌面浏览器测试（Windows + Chrome）— 本次已用自动化验证 ✅

### 设置密码
- [x] 首次进设置页（没设过密码）→ 不出现锁屏，页面正常，密码区标题为「设置访问密码」
- [x] 填「新密码」+「再输一次」一致（≥4 位）→ 点「设置密码」→ 提示已保存，标题变「修改访问密码」
- [x] 两次新密码不一致 → 报错「两次输入的新密码不一致」
- [x] 新密码少于 4 位 → 报错「密码至少 4 位」

### 锁屏解锁
- [x] 设过密码后刷新/重进设置页 → 出现锁屏遮罩，挡住整个页面
- [x] 输错密码 → 提示「密码不对」，仍锁着
- [x] 输对密码 → 遮罩消失，进入设置页
- [x] 锁屏点「← 返回游戏」→ 直接回首页（孩子误入能自己退出，不需要密码）

### 修改密码
- [x] 密码区填「当前密码」错 + 新密码 → 报错「当前密码不对」
- [x] 「当前密码」对 + 新密码一致 → 修改成功，旧密码失效、新密码生效

### 忘记密码 → 家长验证
- [x] 锁屏点「忘记密码？」→ 展开算术题（两位数 × 两位数）
- [x] 算错 → 提示「答案不对」，换一道新题，仍锁着
- [x] 算对 → 解锁进入，密码区变「重设访问密码」（不需要当前密码）
- [x] 在重设模式填新密码 → 保存成功，旧密码失效

### 取消密码保护
- [x] 「当前密码」对 → 点「取消密码保护」→ 提示已取消，标题回到「设置访问密码」，下次进页不再锁屏
- [x] F12 Console 无红色错误
