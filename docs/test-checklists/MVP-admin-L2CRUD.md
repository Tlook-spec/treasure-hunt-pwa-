# 手动测试 Checklist：L2 点位 CRUD（M08）

## 前置条件

先进入某个 L1 探险的详情页（M07 已验证）。

---

## 桌面浏览器测试（Windows + Chrome）

### 点位列表
- [ ] 打开任意探险详情 → 「点位列表」区域显示「还没有点位」空状态
- [ ] 「➕ 添加点位」按钮可点击

### 新建点位
- [ ] 点「添加点位」→ 切换到点位编辑面板，标题显示「第 1 站（新建）」
- [ ] 序号框显示「1」（只读）
- [ ] 4 个字段可编辑：点位名称、家长备忘、本站扫码成功提示、下一点提示
- [ ] 名称留空保存 → 弹 alert，不保存
- [ ] 填入名称后保存 → 返回详情页，点位列表出现新行
- [ ] 新行显示：序号圆圈、点位名称、题目数 0、6 位数字码
- [ ] 再建第 2 个点位 → 序号显示「2」

### 编辑点位
- [ ] 点位行「✏️ 编辑」→ 切换到编辑面板，4 个字段填有当前值
- [ ] 修改名称保存 → 列表更新
- [ ] 编辑时 order 和 code 不变（序号框只读）

### 上移/下移
- [ ] 新建 3 个点位（A、B、C）→ 列表顺序 1:A 2:B 3:C
- [ ] B 行「↑」上移 → 顺序变为 1:B 2:A 3:C
- [ ] C 行「↓」无法再下移（按钮 disabled）
- [ ] 第 1 个点位「↑」按钮为 disabled

### 复制点位
- [ ] 点某行「📋 复制」→ 列表末尾出现新行，名称为「原名（副本）」
- [ ] 副本的 6 位码与原点位不同（新生成）
- [ ] 副本的 order = 当前最大 order + 1
- [ ] 副本保留原来的 questionIds（题目数相同）

### 保存二维码
- [ ] 点某行「🔲 二维码」→ 弹出二维码弹窗
- [ ] 弹窗显示：点位名称、QR 码图片、6 位数字（大字）、说明文字
- [ ] 点「关闭」或遮罩 → 弹窗关闭
- [ ] 点「💾 下载 PNG」→ 浏览器下载一张以「点位名-数字码.png」命名的图片
- [ ] 下载的 PNG 可以用手机相册打开，二维码可被微信/支付宝扫码识别（显示 6 位数字）

### 删除点位
- [ ] 点「🗑️」删除 → 弹出 window.confirm 对话框，包含点位名称
- [ ] 点「取消」→ 不删除
- [ ] 点「确定」→ 点位消失，列表更新
- [ ] 删除后剩余点位的序号不变（order 字段值保持，但可能不连续）

### 题目绑定（先在题库管理里随便加 2-3 道题）
- [ ] 进入点位编辑 → 题目区显示「暂无题目」
- [ ] 点「从题库添加题目」→ 弹出选题面板
- [ ] 面板内显示题库所有题目（每道题含学科标签、难度星、题干）
- [ ] 已绑定到当前点位的题目不在候选列表里
- [ ] 按学科筛选 → 列表过滤
- [ ] 按难度筛选 → 列表过滤
- [ ] 勾选 2 道题 → 底部显示「已选 2 道题」
- [ ] 点「确认添加」→ 题目出现在点位编辑的题目列表
- [ ] 题目显示：#序号、学科标签、难度星、题干前 40 字、上移/下移/移除按钮
- [ ] 点「取消」→ 不添加
- [ ] 保存点位 → 返回详情页，点位行显示「2 道题」

### usedCount 验证（核心业务规则）
- [ ] 添加题目前先记录该题在题库的 usedCount 值（F12 → IndexedDB → questions 表查看）
- [ ] 添加该题并**保存点位** → 题库里该题的 usedCount += 1
- [ ] 从点位里「移除」该题并保存 → usedCount **不减**（保持上一步的值）
- [ ] 重复添加同一题到另一个点位 → usedCount 再 +1

### 题目顺序
- [ ] 绑定 3 道题 → 显示 #1 #2 #3
- [ ] 点 #2 的「↑」→ 变为 #2 #1 #3 的顺序
- [ ] 保存并重新编辑 → 顺序持久化

### 取消/返回
- [ ] 编辑点位时点「← 返回详情」→ 返回详情页，不保存修改
- [ ] 编辑时点「取消」→ 同上

### 题库为空时
- [ ] 题库没有题目 → 打开选题面板 → 显示「题库还没有题目」提示

### Console 检查
- [ ] 全程无红色错误

## iPad Safari 测试

- [ ] 打开 `https://tlook-spec.github.io/treasure-hunt/admin/#/levels`
- [ ] 进入探险详情 → 点位列表可见
- [ ] 添加点位、填表、保存 → 正常
- [ ] 选题面板可以打开并勾选
- [ ] 上移/下移按钮可点击

## 注意事项

- **usedCount** 在保存点位时才写入，仅增不减（消耗式设计）
- 删除点位时，关联题目的 usedCount **不减**
- 若 IndexedDB 需要重置：DevTools → Application → IndexedDB → `treasure-hunt-admin-db` → Delete database
