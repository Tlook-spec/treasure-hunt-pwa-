# 功能名：游戏端 JSON 整库覆盖导入（M18）

## 前置准备

- 在编辑端至少建 1 个探险 + 几道题目，执行「整库导出」，得到 `.json` 文件
- 通过 AirDrop / 邮件 / 微信把该 JSON 文件发到 iPad

---

## 桌面 Chrome 测试

**正常导入流程**
- [ ] 打开 play/pages/settings.html
- [ ] 当前数据显示 0 个探险、0 道题目、上次导入「尚未导入」
- [ ] 点「📥 完整覆盖导入」→ 文件选择对话框打开
- [ ] 选择有效 JSON 文件 → 弹「将导入 N 个探险…无法撤销」确认弹窗
- [ ] 点「取消」→ 弹窗关闭，导入未发生，数据不变
- [ ] 再次点导入按钮 → 选文件 → 点「确定导入」→ 成功卡片「🎉 导入成功！」出现
- [ ] 成功卡片显示「成功导入 N 个探险、X 道题目」数量正确
- [ ] 统计区「上次导入时间」显示刚刚的 YYYY-MM-DD HH:MM（不再是「尚未导入」）
- [ ] 统计区探险数/点位数/题目数更新为导入后的值
- [ ] 点「🏠 返回首页」→ 跳转首页
- [ ] 再点「开始游戏」→ 探险选择页，能看到导入的探险卡片（不再显示空状态）
- [ ] F12 Console 全程无红色报错

**无效文件测试**
- [ ] 选一个普通 .txt 文件（改扩展名为 .json）→ 弹「文件校验失败 + 错误文案」→ 终止导入
- [ ] 选一个 JSON 但缺少 schemaVersion → 弹「这不是有效的寻宝游戏数据文件」
- [ ] 选一个 schemaVersion 不是 "1.0" 的 JSON → 弹「该文件由不兼容的 App 版本创建」
- [ ] 以上情况均无崩溃，弹窗点「好的」关闭后可重新导入

**重复导入**
- [ ] 导入成功后，刷新页面 → 统计数据依然正确，「上次导入时间」依然显示（localStorage 持久）
- [ ] 成功导入后再导入一次不同的 JSON → 流程正常，数据被覆盖

## 备注

- `lastImportAt` 写在 localStorage，刷新页面不会丢失（IndexedDB 也不会丢失）
- photos 表不会被导入清除（已拍的合影暂时失去 session 关联，但照片数据保留）
- 导入失败（importFull 事务回滚）时，原有数据应保持完整（原子性保证）
