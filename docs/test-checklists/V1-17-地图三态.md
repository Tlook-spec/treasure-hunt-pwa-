## 桌面浏览器测试（Windows + Chrome，Live Server）

### 前提：使用有地图的探险（已上传底图 + 设置点位坐标），至少 3 个点位

**正向流程 — 从头玩**

- [ ] 开场故事页：地图显示但不画任何标记（所有站未发现，底图可见）
- [ ] 扫到第 1 站 → 发现屏出现地图：第 1 站空心圆 + 浮现动画（discovered），其余不画
- [ ] 发现屏 → 点「继续」开始答题，答题屏出现 🗺️ FAB
- [ ] 点 FAB → 地图覆盖层：第 1 站空心圆（discovered），其余不画
- [ ] 第 1 站全部答完 → 进拍照页：地图内嵌，第 1 站盖红星+金字（completed），其余不画
- [ ] 继续扫第 2 站 → 发现屏：第 1 站盖章、第 2 站空心圆浮现
- [ ] 答题屏 FAB → 第 1 站盖章、第 2 站空心圆
- [ ] 第 2 站答完拍照页 → 第 1、2 站盖章，其余不画

**跨站状态一致性**

- [ ] 同一站：发现屏、答题屏 FAB 看到的该站状态相同（均为 discovered）
- [ ] 同一站：拍照页地图和答题屏 FAB 看到的该站状态相同（拍照页盖章，答题屏 FAB 在答题时为 discovered，离开后不用再验）
- [ ] 已完成站在每一处均显示红星+金字，不会错显成空心圆

**续玩场景（关键验证）**

- [ ] 答完第 1 站、进入第 2 站提示页后，关闭 Live Server（模拟退出）
- [ ] 重新打开，从「继续上次探险」进入 hint.html
- [ ] hint.html FAB → 第 1 站仍为红星盖章（从 IndexedDB 恢复，非内存状态）
- [ ] 继续扫第 2 站，发现屏 → 第 1 站盖章、第 2 站空心圆（完全正确）

**无地图探险**

- [ ] 无地图探险：发现屏不显示地图区块，答题屏无 FAB，拍照页无地图（对照组，不出错）
