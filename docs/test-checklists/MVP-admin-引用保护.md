# 功能名：M14 删除引用保护

## 前置条件

- 编辑端已有 1 个探险，含 1 个点位，点位绑定了 1 道题目

---

## 有引用时的拒绝流程

- [ ] 进入「题库管理」，找到已被点位引用的题目，点「🗑️ 删除」
- [ ] 弹窗标题显示「❌ 无法删除」
- [ ] 弹窗内容显示：「这道题正在被以下 N 个点位使用：」
- [ ] 弹窗内容显示引用列表（带探险名 + 第 X 站 + 点位名），格式：`探险名 · 第 N 站「点位名」`
- [ ] 弹窗内容显示：「请先去这些点位中移除引用，再回来删除。」
- [ ] 弹窗只有「我知道了」按钮（无「确认删除」按钮）
- [ ] 点「我知道了」或「✕」→ 弹窗关闭，题目未被删除
- [ ] 点弹窗遮罩区域 → 弹窗关闭
- [ ] F12 Console 无红色错误

## 无引用时的正常删除流程

- [ ] 找到没有被任何点位引用的题目，点「🗑️ 删除」
- [ ] 弹窗标题显示「🗑️ 确认删除」
- [ ] 弹窗内容显示：「确定删除题目「xxx」吗？」
- [ ] 弹窗有「取消」和「确认删除」两个按钮
- [ ] 点「取消」→ 弹窗关闭，题目保留
- [ ] 点「确认删除」→ 题目被删除，列表刷新
- [ ] 删除后该题从列表消失

## 解除引用后再删除（完整流程）

- [ ] 进入 L2 点位编辑，移除绑定的那道题，保存点位
- [ ] 回到题库，找到刚才解除引用的题目
- [ ] 该题的引用数 badge 显示「🔵 0次」
- [ ] 点「删除」→ 弹「确认删除」弹窗（不再显示引用拒绝）
- [ ] 确认删除 → 题目成功删除

## 多点位引用的情况

- [ ] 同一道题被 2 个以上点位引用时，拒绝弹窗列出所有引用点位
- [ ] 不同探险的点位引用时，引用列表分别显示各自的探险名
