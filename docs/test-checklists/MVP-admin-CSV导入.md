# 手动测试 Checklist：CSV 批量导入题目（M10）

## 桌面浏览器测试（Windows + Chrome）

### 入口与页面展示
- [ ] 进入「题库管理」→ 点「📥 CSV 导入」→ 切换到导入页面（标题「📥 批量导入题目」）
- [ ] 页面显示橙色警告框，内容包含「UTF-8」「GBK」「另存为」「CSV UTF-8（逗号分隔）」等关键字
- [ ] 显示两个教学截图占位（后续换成真实截图）
- [ ] 步骤 1 有「📥 下载模板（带表头和例子）」按钮
- [ ] 步骤 2 有「📁 选择文件」按钮，右侧显示「未选择文件」
- [ ] 「✅ 确认导入」按钮处于禁用状态（灰色，不可点）
- [ ] F12 Console 无红色错误

### 步骤 1：下载模板
- [ ] 点「📥 下载模板」→ 浏览器下载 `题目导入模板.csv`
- [ ] 用 Excel/记事本打开 → 第一行为表头：学科、难度、年龄段、题型、题干、选项A–D、正确答案、提示、解析
- [ ] 第 2 行是选择题例子，第 3 行是判断题例子
- [ ] 中文显示正常（无乱码），说明 BOM 写入正确

### 步骤 2：上传 UTF-8 文件（正常流程）
- [ ] 用 Excel 打开模板，**另存为「CSV UTF-8（逗号分隔）」**，加入 2-3 道自己的题目
- [ ] 点「📁 选择文件」→ 选择刚保存的 CSV → 文件名显示在按钮右侧
- [ ] 预览区出现（标题「预览（前 5 行数据）：」）
- [ ] 预览表格显示列名和前几行数据，中文正常
- [ ] 无红色乱码警告
- [ ] 「✅ 确认导入」按钮变为可点击状态（蓝色）

### 确认导入
- [ ] 点「✅ 确认导入」→ 按钮短暂显示「导入中…」
- [ ] 导入完成 → 显示「✅ 成功导入 X 道题目！」（绿色结果框）
- [ ] 点「← 返回题库」→ 题库列表刷新，刚导入的题目出现在列表中
- [ ] 导入的题目标签（学科/难度/题型）与 CSV 内容一致
- [ ] 导入的题目 usedCount 徽章显示 🔵 0次

### UTF-8 GBK 乱码处理（Task 1 改为从严字段校验，不再做乱码检测）
- [ ] 用 Excel 打开模板，**直接另存为普通「CSV（逗号分隔）」（不选 UTF-8）**，填中文题目
- [ ] 点「📁 选择文件」→ 选择这个 GBK 格式的 CSV
- [ ] 预览区出现，数据可能显示乱码
- [ ] 点「✅ 确认导入」→ 导入逐行校验：学科字段乱码后不在白名单（数学/语文/英语/科学/常识），报「未知学科」错误
- [ ] 结果显示「⚠️ 失败 X 道」+ 每行错误详情，正确行（如果有）仍导入成功

### 错误行处理
- [ ] 在 CSV 里故意加几行错误数据：
  - 空题干 → 成功后报「第X行：题目内容不能为空」
  - 无效学科（如「政治」）→ 报「学科无效」
  - 正确答案填「E」→ 报「正确答案无效」
- [ ] 错误行不影响其他正确行的导入
- [ ] 导入结果显示「✅ 成功 X 道，⚠️ 失败 Y 道（详见下方）」（橙色结果框）
- [ ] 下方列出所有错误的行号和原因

### 注释行跳过
- [ ] 模板里有一行以 `#` 开头的提示行 → 导入时自动跳过，不报错也不计入失败

### 返回与状态重置
- [ ] 导入完成后点「← 返回题库」→ 回到题库列表
- [ ] 再次点「📥 CSV 导入」→ 导入页面重置：文件名「未选择文件」、预览区隐藏、按钮禁用

### Console 检查
- [ ] 全程无红色错误

## 注意事项

- **BOM**：模板下载带 UTF-8 BOM（`﻿`），Windows Excel 才能正确识别中文列名
- **乱码检测**：通过检查解析结果是否含 U+FFFD（`�`）替换字符来判断，无需编码检测库
- **正确答案格式**：CSV 中填字母 A/B/C/D，与 M09 题库 `correctAnswer` 字段格式一致
- **判断题**：options 固定['对','错']，CSV 填的 选项A/选项B 会被忽略，以固定选项为准
- **usedCount**：导入时写 0 但不作为统计来源，引用数由系统实时扫描 Point.questionIds 计算
