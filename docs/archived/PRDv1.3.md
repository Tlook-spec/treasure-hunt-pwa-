# 寻宝游戏 PWA - 产品需求文档（PRD）

> 本文档定义产品的完整需求。Claude Code 在实现具体功能时，请查阅本文档的对应章节。
>
> **当前版本：v1.3**（详见 §10 文档更新记录）

---

## 1. 产品概述

### 1.1 一句话定位

**一个轻量级的儿童户外解谜游戏，包含两个独立网站**：
- **编辑端**（家长用）：普通网页，在电脑/平板/手机浏览器里设计关卡和题库
- **游戏端**（孩子用）：iPad 上的离线 PWA，扫码答题、户外探险

### 1.2 使用场景

家长在电脑上设计探险（关卡）、批量录入题库、生成二维码并打印；周末把二维码贴到小区/公园打卡点；孩子拿 iPad 按顺序找点、扫码、答题、拍照纪念，全程离线。

### 1.3 核心特性

- **双站点架构**（v1.3 重大调整）：
  - 编辑端：普通网页，**不做 PWA**，浏览器打开即用
  - 游戏端：完整 PWA，**完全离线** + 安装到主屏幕
  - 数据通过 JSON 文件手动同步（编辑端导出 → AirDrop/邮件 → iPad 导入）
- **三层结构**（v1.3 重新设计）：
  - **L1 探险（Level）**：一次完整的户外活动，含 3-5 个 L2 点位
  - **L2 点位（Point）**：一个物理打卡点，1 个二维码 + 多道题目
  - **L3 题目（Question）**：一道独立的题，被 L2 引用
- **完全离线**：游戏端户外无网络也能玩
- **顺序解锁**：必须按家长设计的顺序通关
- **自定义内容**：家长可自由创建探险、添加题目、设计提示
- **MVP 单人优先**：MVP 单人模式跑通；V1 加多人模式

### 1.4 设计哲学

- **MVP > 完美**：先做单人闭环，验证「孩子是否真喜欢」，再扩展
- **简单 > 全面**：自用 App，砍掉所有非必要的工程化设计
- **温和 > 严苛**：答错有容错、求助有机会、孩子不会「卡死」
- **数据可控 > 体验丝滑**：电脑端是 source of truth，iPad 是消费端
- **JSON 文件就是数据库**：编辑端的浏览器只是工作台，JSON 是真正的数据持久层

## 2. 用户角色

### 2.1 家长（编辑者）

**画像**：4-12 岁孩子的父母

**典型任务**：
- 在电脑上批量录入题库（用 Excel 整理后 CSV 导入编辑端）
- 在编辑端设计 L1 探险、配置 L2 点位、绑定 L3 题目
- 生成二维码并打印
- 出门前实地贴二维码
- 导出 JSON 数据，传到 iPad 同步
- 游戏过程中陪伴孩子、应急处理

### 2.2 孩子（玩家）

**画像**：4-12 岁儿童

**典型任务**：
- 启动游戏端 PWA、选择探险
- 看提示找下一个点位
- 扫描二维码（或输入 6 位短码）
- 答题（一个点位多道题，连续答）
- 答完所有题后拍一张通关合影
- 通关后看成绩

## 4. 详细功能说明

### 4.1 编辑端（admin/）

> **编辑端是普通网页，不做 PWA**。家长在浏览器里访问 `https://你的用户名.github.io/treasure-hunt/admin/` 即可使用。
> 
> **iPad 浏览器访问编辑端也可以**（响应式适配），但 iPad PWA（游戏端）里没有编辑功能。

#### 4.1.1 首页 / 导航

- 顶部 logo + 标题「寻宝游戏 - 编辑端」
- 左侧导航栏（桌面端固定 / 移动端汉堡菜单）：
  - 📝 探险（L1）管理
  - 📚 题库管理
  - 📥 导入 JSON
  - 📤 导出 JSON
- 主区域：默认显示「探险管理」

#### 4.1.2 L1 探险管理

**探险列表**：
- 卡片列表显示所有探险
- 每张卡片：探险名、点位数、上次编辑时间
- 操作：新建、编辑、复制、删除、单独导出

**新建 / 编辑 L1 表单**：
- 探险名称（必填）
- 简介（选填）
- 推荐人数（1-6，V1 用）
- 推荐年龄段（4-6 / 7-9 / 10-12）
- 主题色（预设几个）
- L1 密码（仅用于游戏端「找不到？」跳过的家长验证）
- 开场故事（V1 选填）
- 结局故事（V1 选填）

#### 4.1.3 L2 点位编辑

**L2 列表**（在 L1 详情页内）：
- 显示该 L1 的所有 L2 点位
- 拖拽排序（桌面端）/ 上移下移按钮（移动端）
- 操作：新建、编辑、复制、删除点位

**L2 编辑表单**：
- **基本信息（仅家长可见）**：
  - 点位序号（自动）
  - 点位名称（如「小区凉亭」）
  - 家长备忘（如「贴在凉亭东侧第三根柱子上」）
- **下一点提示（孩子可见）**：文本框，支持多行
- **L2 包含的题目**：
  - 显示已绑定的题目列表（来自题库）
  - 「➕ 从题库添加题目」按钮：弹出题库选择面板（含筛选条件）
  - 「🎲 随机抽 N 道」按钮（V1）：按标签随机抽，避开已用 ≥ 2 次的
  - 每道题旁有「移除」按钮
  - 拖拽 / 上移下移排序（决定答题顺序）

#### 4.1.4 题库管理（L3）

**题库主页**：
- 顶部筛选条：学科、难度、年龄段、使用次数（0 / 1 / 2+）
- 列表显示所有题目
- 每行显示：题干前 30 字、标签、难度、`usedCount`
- 使用次数显示：
  - 🔵「0 次」
  - 🟡「1 次」
  - 🟠「2+ 次」（建议替换）
- 操作：新建、编辑、删除、复制、CSV 导入、CSV 导出、JSON 导出
- **批量操作**：选中多道题 → 「批量删除使用 ≥ 2 次的题目」

**题目数据结构**：见 §6.3。

**MVP 题型**：单选、判断
**V1 题型**：多选、填空 + 题目插图
**V2 题型**：拖拽连线、图片选择

**题目「消耗式」使用模型**：

> 你的工作流：一次性导入一大批题（如 100 道），创建 L1 时从大池子里挑没用过的，用了 2 次后整体清理。

- 题目绑定到 L2 时，`usedCount += 1`
- 题目从 L2 移除时，**不减**（已消耗）
- 题库默认按 `usedCount` 升序显示（未用过的优先）
- 提供一键操作：「删除所有 usedCount ≥ 2 的题目」

##### CSV 批量导入流程

**步骤 1：下载模板**
- 模板列：学科、难度、年龄段、题型、题干、选项A、选项B、选项C、选项D、正确答案、提示、解析
- UTF-8 with BOM 编码

**步骤 2：用 Excel 编辑** → 保存为 CSV（UTF-8）

**步骤 3：上传导入**
- 使用 PapaParse 自动检测编码
- 预览前 5 行
- 确认导入
- 显示「成功 X 道，失败 Y 道（行号 + 原因）」

#### 4.1.5 二维码生成与打印

**编码方式（v1.3 简化）**：

每个 L2 点位生成一个**6 位随机数字码**（如 `482931`）。

- 这个码同时用于：
  - 生成二维码（QR Code 内容就是这 6 位数字）
  - 让孩子手动输入（扫不出时）
- **不需要 HMAC，不需要 levelId+pointId 拼接**
- 验证逻辑：扫到的码去 IndexedDB 查 Point 表，找到对应点位即可

**Point 创建时**：随机生成 6 位数字码，检查与已有码是否冲突，冲突则重新生成（自用场景几乎不会冲突）。

**生成流程**：
- L2 编辑页有「预览二维码」按钮
- L1 详情页有「生成全部二维码」按钮
- 单个二维码可保存 PNG

**打印流程**：
- L1 详情页有「🖨️ 打印二维码」按钮
- 触发 `window.print()` + CSS 打印样式
- 浏览器原生「打印 / 另存为 PDF」对话框
- 每页 2-4 个二维码，每个码下方：
  - 大字：第 X 站
  - 中字：点位名称
  - 大字：6 位短码（如「482931」）
  - 小字：家长备忘

#### 4.1.6 数据导入导出

**核心架构**：编辑端浏览器的 IndexedDB 是工作台，**JSON 文件是真正的数据库**。

##### 导出

**导出选项**：
- **全部数据导出**：所有探险 + 全部题库（一份完整备份）
- **单 L1 导出**：包含该 L1 + 它引用的所有题目
- **仅题库导出**：不含探险
- **CSV 导出**：仅题库

**文件格式**：单个 JSON 文件，**不含图片**。

##### 导入

**MVP 策略**：「完整覆盖」模式（默认）+ 二次确认

- 导入时弹「这将替换当前所有数据，确定吗？」
- 确认后清空 IndexedDB，写入 JSON 数据

**为什么用覆盖而非合并**：
- 工作流是「JSON 文件是权威数据」
- 编辑端只是工作台
- 覆盖是 single source of truth 的自然实现

**V1**：可选「追加」或「智能合并」模式

##### 典型工作流

**新电脑/换浏览器**：
1. 打开编辑端网页
2. 导入之前的 JSON 备份
3. 继续工作

**编辑 → iPad 同步**：
1. 编辑端完成所有编辑
2. 「📤 导出全部数据」→ 下载 JSON
3. AirDrop / 邮件 / 微信 / iCloud Drive 传到 iPad
4. JSON 保存到 iPad「文件」App
5. 在 iPad 上打开游戏端 PWA → 进入「设置」/ 「同步数据」 → 「📥 导入数据」
6. 弹「检测到正在进行中的游戏，导入会清除进度」**如果有未完成游戏**
7. 选择文件 → 导入完成

> **iPad 上的游戏端没有「编辑」入口**——导入数据按钮可以放在游戏端的「设置」或开始页角落，不需要密码保护。

##### 图片处理策略

**MVP**：JSON 完全不含图片
- 题目无插图
- 拍的合影存在 iPad 本地，不通过 JSON 同步（用户自己长按存到相册）

**V1**：JSON 扩展为 zip 包（包含题目插图）

#### 4.1.7 删除 + 撤销

**简化方案（v1.3）**：

- 删除前：「确定要删除「凉亭」吗？」二次确认弹窗
- 删除后：底部 toast「已删除『凉亭』 [撤销]」5 秒可撤回
- 5 秒后真正删除（IndexedDB 物理删除）

**不做软删除 / 最近删除**（v1.2 那个机制砍掉）。如果家长真的误删超过 5 秒，从 JSON 备份恢复即可。

### 4.2 游戏端（play/）PWA

#### 4.2.1 启动与首屏

**启动页（MVP 单人版）**：
- App 图标 + 「寻宝游戏」标题
- 大按钮「开始游戏」
- 小字「设置」（角落，含「导入数据」入口）

**探险选择页**：
- 卡片列表显示所有探险
- 每张：探险名、点位数、上次游玩
- 点击卡片进入「开始冒险」流程

#### 4.2.2 开始游戏（MVP 单人版）

- 显示探险名 + 大按钮「开始冒险！」
- 点击直接进入第 1 个 L2 点位的提示页（不需要起始码）

**V1 多人版会插入**：
- 人数选择 → 玩家创建 → 队名设置 → 开始

#### 4.2.3 游戏主流程（每个 L2 点位）

**1. 提示展示**
- 大字：当前是第几站
- 提示文本
- 大按钮「我找到了！扫码！」

**2. 扫码 / 输码**
- 全屏摄像头预览 + 扫码取景框
- 底部「输入数字码」按钮（弹 6 位数字输入框）
- 底部「找不到？」按钮（弹 L1 密码输入 → 跳过当前点）

**3. 扫码 / 短码结果**
- **当前点位**：「叮咚」+ 撒花 → 进入答题
- **其他点位的码**：友好提示「这不是你现在要找的点哦！再看看提示～」
- **完全无效的码**：提示「这不是寻宝游戏的码哦」

**4. 多题连答（v1.3 新增）**

一个 L2 点位有多道题，依次答：

- 显示「这一站有 3 道题，第 1 / 3 题」
- 每道题：题目 + 选项 + 求助按钮
- 答题反馈机制同 §4.2.4
- 一道题答完进入下一道
- 所有题答完后，进入拍照环节

**5. 答题反馈（每道 L3 题）**
- **答对**：「叮咚」+ 绿色对勾 + 进入下一题
- **第 1 次答错**：温和提示「再想想？」可重选
- **第 2 次答错**：温和提示 + 显示题目提示（如有） + 可重选
- **第 3 次答错（保底）**：显示正确答案 + 「没关系，下次记住啦！」+ 自动通过（0 分）

**6. L2 通关拍合影（v1.3 调整：每 L2 一张照）**

**核心**：每个 L2 点位**只在完成时拍一张**，不是每道题都拍。

- 一个 L2 所有题答完后，显示「📸 拍一张通关合影（可选）」按钮 + 「跳过」按钮
- 点拍照：调系统相机
- 拍完照片显示，下方提示「**长按图片可保存到系统相册**」
- 同时存到 IndexedDB（每 L2 一张，最多约 25 张 / 整个 App）
- 照片压缩：1280px 长边，JPEG quality 0.7
- 拒绝相机权限或跳过：直接进入下一步

**7. 下一站衔接**
- 不是最后一站：显示下一站提示，大按钮「前往下一站」
- 是最后一站：直接进入通关庆典页

#### 4.2.4 求助机制

- 答题页右上角「求助 N」按钮
- 初始 3 次
- 点击：
  - 次数 -1
  - 弹动画「快去问爸爸妈妈！」
  - **标记本题 usedHelp = true**

#### 4.2.5 积分系统（MVP 单人版）

只计总分，不区分玩家。

| 情况 | 得分 |
|------|------|
| 一次答对（未用求助） | 10 分 |
| 答错 1 次后答对（未用求助） | 5 分 |
| **曾用求助**最终答对 | 3 分 |
| 第 3 次答错保底通过 | 0 分 |

V1 加：难度 2 星 ×1.2、难度 3 星 ×1.5。

#### 4.2.6 通关庆典（MVP 极简版）

**单屏**：
- 撒花动画
- 大字「🎉 通关啦！」
- 总分：「X 分」
- 用时：「Y 分钟」
- 拍的所有 L2 通关合影（横向滑动展示，最多 5 张）
- 大按钮「返回首页」

V1 加完整版（多屏 + 投票 + 颁奖 + 保底奖）。

#### 4.2.7 暂停与续玩

**自动保存**：每次扫码、答题、拍照后自动保存进度到 IndexedDB

**主动暂停**：游戏中右上角「暂停」按钮 → 继续 / 退出 → 退出后状态保留

**续玩**：
- 启动游戏端时检查未完成游戏
- 有 → 弹「上次的游戏还没玩完，要继续吗？」
- 继续：恢复到上次状态（直接读当前 L1 数据）
- 放弃：清除存档

**导入数据保护（v1.3 新增）**：
- 导入 JSON 前检查是否有未完成游戏
- 有 → 弹「检测到正在进行中的游戏，导入会清除进度，确定吗？」
- 简单粗暴，避免数据混乱

#### 4.2.8 应急通关（家长操作）

- 游戏中长按屏幕右上角 3 秒 + 输 L1 密码 → 进入应急菜单
- 选项：
  - 跳过当前点（不计分）
  - 跳到指定点
  - 强制通关
  - 重置游戏

## 7. 边界情况与异常处理

### 7.1 编辑端

- **删除 L1 / L2 / 题目**：弹二次确认 → 删除 → 5 秒 toast 撤销窗口
- **题目被 L2 引用时删除**：提示「该题被 X 个 L2 使用，确认删除？」
- **二维码数字码冲突**：生成时检查全表，冲突则重新生成
- **L1 密码丢失**：编辑 L1 时可查看 / 修改
- **CSV 导入错误**：单行错误不影响其他行；显示成功/失败统计 + 失败原因
- **CSV 编码不对**：提示「请用 UTF-8 保存」+ 教学截图链接

### 7.2 游戏端

- **扫到其他 L2 的码**：友好提示「这不是现在要找的点哦」
- **扫到完全无效的码**：「这不是寻宝游戏的码哦」
- **二维码物理损坏**：「找不到？」按钮 → 输 L1 密码 → 跳过
- **扫码不清晰**：「输入数字码」按钮 → 输 6 位数字
- **答题中切换 App**：恢复后回到答题状态
- **拍照权限拒绝**：不阻塞，直接进下一步
- **iOS Safari 切后台黑屏**：监听 visibilitychange，切回时重新初始化摄像头
- **扫码后立即关闭摄像头流**，1 秒后才能重启
- **设备旋转**：游戏中竖屏 ↔ 横屏自动适配但不强制
- **电量过低**：提示「iPad 没电啦，建议先充电」

### 7.3 数据安全

**iOS 清理 PWA 数据风险**：
- 提示用户定期开启游戏端（保持「活跃」状态）
- 重要数据在编辑端导出 JSON 备份
- 拍的照片提示「长按保存到相册」

**编辑端浏览器数据丢失**：
- 用户自己保管 JSON 备份
- 多设备共用：每次导出 / 导入 同一份 JSON
- 提示用户：「JSON 文件就是你的数据库，请妥善保管」

### 7.4 数据导入冲突

- **导入 JSON 时有未完成游戏**：弹「检测到正在进行中的游戏，导入会清除进度，确定吗？」
- **导入 JSON 版本不兼容**：检查版本号，不兼容时友好提示

### 7.5 Service Worker 更新（游戏端）

- cache-first 策略
- 启动时后台检查新版本
- 有新版本顶部显示「点此刷新」按钮
