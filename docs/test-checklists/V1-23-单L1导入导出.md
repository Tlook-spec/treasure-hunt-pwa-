# 功能名：V1-23 单 L1 导出 + 追加导入

> 编辑端 L1 详情页新增「📤 导出这个探险」按钮，导出单个 L1 子树（JSON exportType='single-level'）。
> 游戏端设置页 + 编辑端导入导出页均支持两种类型：full（整库覆盖）和 single-level（追加/覆盖单 L1）。

---

## 编辑端测试（桌面 Chrome + iPad Safari）

### 单 L1 导出

- [ ] 进入任意 L1 详情页 → 工具区域有「📤 导出这个探险」按钮
- [ ] 点击导出 → 弹出下载文件，文件名含探险名 + 日期（如 `treasure-hunt-丛林探险-2026-06-23.json`）
- [ ] F12 看下载的 JSON：
  - `exportType` = `"single-level"`
  - `data.levels` 只有 1 个（该探险）
  - `data.points` 是该探险的点位（非其他探险点位）
  - `data.questions` 是这些点位引用的题目（无多余题目）
  - `schemaVersion` = `"1.1"`

### 编辑端导入页（import-export.html）支持两种文件

- [ ] 选择一个 `single-level` 文件：
  - 步骤 2 预览区显示「探险《名字》· X 个点位、Y 道题」（不是「X 个探险」）
  - 点「确认导入」弹窗显示该探险是否已存在（已有 → 「已存在，覆盖?」；没有 → 「将添加探险」）
  - 弹窗内有「注意：题目会按导入版本更新…」小字
- [ ] 确认覆盖已存在的探险 → 导入成功，其余探险**仍然在**（没被清空）
- [ ] 导入一个全新探险（ID 不存在）→ 追加进去，其余不动
- [ ] 选择旧的「整库 full」文件 → 走原来的整库覆盖流程（弹窗仍显示「确认替换所有数据」）
- [ ] 旧 1.0 整库文件导入 → 仍能导入（V1-19 放宽校验没被破坏）
- [ ] F12 Console 无红色错误

## 游戏端测试（按 §5.7 三段结构）

### 桌面浏览器测试（Windows + Chrome，Live Server）

- [ ] 设置页选择 `single-level` 文件 → 弹「添加/覆盖」确认弹窗（不是「整库替换」弹窗）
- [ ] 确认后：
  - 该探险被导入，设置页显示的探险数正确
  - **其余探险还在**（核心！不能被清空）
- [ ] 再次导入同一文件（同 ID 的探险）→ 弹「已存在，覆盖?」→ 确认后更新
- [ ] 该探险有未完成存档时导入 → 额外弹「有没玩完的存档，覆盖会影响续玩」提示
- [ ] 选择整库 `full` 文件 → 走原来整库流程（「完整替换所有数据」提示），不受影响
- [ ] 旧 1.0 备份（full）导入 → 仍能通过校验
- [ ] F12 Console 无红色错误
