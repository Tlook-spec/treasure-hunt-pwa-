# 寻宝游戏 PWA - 阶段 0-1 任务清单

> 这是你接下来 **2 周** 的"作战手册"。每个任务都是**可以直接复制粘贴**给 Claude Code 的指令。
> 
> **使用方法**：
> 1. 按顺序执行任务（前置任务未完成不要跳）
> 2. 每个任务的【操作指令】部分直接复制给 Claude Code
> 3. 完成后用【验证清单】检查是否真的做完了
> 4. 在任务前的 `[ ]` 打钩 `[x]` 表示完成

---

### [ ] T01. 注册 GitHub 账号

**前置**：无
**预计时间**：15 分钟
**类型**：账号准备

   - 验证邮箱
4. 注册完成后登录

**验证清单**：
- [ ] 邮箱已验证
- [ ] 能正常登录 https://github.com
- [ ] 记下你的用户名（后面要用）

**遇到问题怎么办**：
- 注册邮件没收到：检查垃圾邮件箱；或者换个邮箱（推荐 Gmail / QQ 邮箱）
- 国内网络访问慢：耐心等待，多刷新几次

### [ ] T02. 安装 VS Code 和 Claude Code

**前置**：T01
**预计时间**：30-60 分钟
**类型**：工具安装

**你要做的事**：

**1. 安装 VS Code**：
- 访问 https://code.visualstudio.com/
- 点击 Download for Mac
- 下载后双击安装

**2. 安装 Claude Code**：
- 打开 Mac 自带的"终端" App（Cmd + Space 搜索"terminal"）
- 访问 https://docs.claude.com/en/docs/claude-code/quickstart 查看最新安装命令
- 复制安装命令到终端运行
- 用你的 Claude Pro 账号登录

**3. 在 VS Code 里启用 Claude Code**：
- 打开 VS Code
- 在 VS Code 内部打开终端（菜单：Terminal → New Terminal）
- 在终端输入 `claude` 看是否能启动
- 如果能看到欢迎界面，说明成功了

**验证清单**：
- [ ] VS Code 能正常打开
- [ ] 在 VS Code 的终端里能运行 `claude` 命令
- [ ] Claude Code 显示已登录你的账号

**遇到问题怎么办**：
- 安装命令报错：把完整错误截图，去 https://docs.claude.com 找官方文档
- 终端找不到 claude 命令：可能需要重启终端，或者环境变量没配好——让 Claude Code（在浏览器版 claude.ai）帮你诊断

### [ ] T06. 在 iPad 上添加到主屏幕

**前置**：T05
**预计时间**：30 分钟
**类型**：测试任务

8. 回到主屏幕，找到刚添加的"寻宝游戏"图标

**验证清单**：
- [ ] iPad 主屏幕上有寻宝游戏图标
- [ ] 点击图标打开后没有 Safari 浏览器栏

### [ ] T07. 扫码功能试水

**前置**：T06
**预计时间**：2-3 小时
**类型**：技术验证

**【复制给 Claude Code 的指令】**

```
现在要做第一个真实的功能试水：扫描二维码。

这个功能是整个 App 最核心的能力之一，必须先验证在 iPad Safari PWA 模式下能稳定工作。

请帮我：

1. 创建一个新页面 test-scan.html，独立于主页面
   - 顶部一个返回按钮
   - 中间一个大按钮"开始扫码"
   - 点击后调起摄像头，全屏扫码
   - 扫到任意二维码后：显示扫到的内容文字
   - 提供"再扫一个"按钮

2. 使用 html5-qrcode 库（开源免费），用 CDN 方式引入（不要 npm，保持纯静态）

3. 处理权限相关的边界情况：
   - 第一次扫码请求摄像头权限
   - 权限被拒绝时友好提示"请允许相机权限才能扫码哦"
   - 不支持的浏览器友好提示

4. 在 index.html 上加一个临时入口"测试扫码功能"，链接到 test-scan.html

5. 更新 PROGRESS.md 记录

要求：
- 代码尽量简单，多加中文注释
- 不要做复杂的样式，能用就行（后面会统一美化）
- 完成后告诉我怎么测试：包括本地测试和 iPad 测试

补充说明：
- 我会先在浏览器本地测试（用 Mac 摄像头）
- 然后 push 到 GitHub Pages 在 iPad 上测试
- iPad 真实测试通过才算这个任务完成
```

**你完成后要做的验证**：

**本地测试（Mac 浏览器）**：
1. 启动本地服务器，打开 test-scan.html
2. 点击"开始扫码"，授权摄像头
3. 用手机生成一个二维码（任何二维码生成网站都行）显示给 Mac 摄像头看
4. 确认能扫到并显示内容

**iPad 真实测试（最重要）**：
1. 把代码 push 到 GitHub Pages（让 Claude Code 教你怎么 push 更新）
2. 在 iPad 主屏幕图标打开 PWA
3. 进入测试扫码页面
4. 用另一个手机（或者 Mac）生成二维码
5. 让 iPad 扫码
6. 确认能正常扫到、显示内容

**关键观察**：
- 扫码识别速度（户外可接受范围 1-3 秒）
- 强光下的识别率（可以拿到窗边试试）
- 摄像头权限提示是否友好

**验证清单**：
- [ ] 本地浏览器测试通过
- [ ] 部署到 GitHub Pages 成功
- [ ] iPad 上从主屏幕打开 PWA，扫码功能正常
- [ ] 扫码识别速度可接受
- [ ] PROGRESS.md 已更新

**这一步的意义**：

**这是整个项目的"技术可行性"验证**。如果这一步走通，说明 PWA 方案完全可行，后面所有功能都不在话下。如果走不通（比如 iPad Safari 死活扫不出码），我们需要重新讨论方案。

在 `docs/` 下创建 `design-tokens.md`，记录所有视觉决策。

**做的决策**：

1. **配色方案**：
   - 去 https://coolors.co/ 或 https://colorhunt.co/ 找一组儿童友好的配色
   - 记录主色、辅助色、强调色、错误提示色、背景色的具体色号

2. **字体**：
   - 中文：系统苹方（PingFang SC）
   - 英文：系统默认（San Francisco）

3. **字号层级**：
   - 大标题、副标题、正文、提示文字的具体 px 值

4. **圆角**：
   - 按钮、卡片、图片的圆角值

5. **间距**：
   - 内边距、元素间距的基础值

6. **整体风格关键词**（选 1-2 个）：
   - 圆润 / 卡通 / 明亮 / 简约 / 清新 / 绘本风

**示例 design-tokens.md**：


## 💡 一些跨任务的提醒

### 关于 git commit 节奏

每完成一个任务（T01、T02……），就让 Claude Code 帮你 commit 一次：

```
请帮我 commit 当前修改，commit message 写：完成 T0X：[任务名]
```

这样以后随时能回到任何一个任务完成时的状态。

### 关于"卡住超过 30 分钟"的处理

如果你某个任务卡了 30 分钟还没解决，**不要硬扛**：

1. 把当前情况写清楚发给 Claude.ai 网页版
