# 寻宝游戏 PWA - 版本路线图

> 本文档对应 PRD v1.3 的版本规划，详细列出 MVP、V1、V2 的功能清单 + 预估开发时间。
> 时间预估基于「零基础 + Claude Code 辅助 + 每天 1-2 小时 / 每周 5-8 小时」的投入强度。
>
> **当前版本：v1.1**（基于 PRD v1.3 重新估时，比 v1.0 缩短 1-2 周）

---

## 📊 总览

```
现在 ──┬── MVP (5-6 周) ──┬── V1 (3-4 周) ──┬── V2 (4-6 周) ──→ 长期维护
       │                  │                 │
       │ 单人能玩          │ 多人 + 完整庆典  │ GitHub 同步
       │ 双站点架构        │ + 投票 + 朗读    │ + 海报 + 雷达
       │ L2 多题 + 合影    │                 │
```

| 版本 | 估时 | 里程碑 |
|------|------|--------|
| **阶段 0+1** | 完成中 | 环境就绪 + 双站点架构调整 + 设计 |
| **MVP** | 5-6 周 | 单人跑通一个 L1（含多 L2 + 多题） |
| **V1** | 3-4 周 | 多人 + 完整庆典 |
| **V2** | 4-6 周 | 自动同步 + 海报 + 雷达 |
| **总计** | **14-18 周（3.5-4.5 个月）** | 完整可用 App |

> 💡 真实工期一般是 1.3-1.5 倍。心理预期 **4.5-6 个月**。

> 🆚 比 v1.0 路线图缩短 1-3 周。原因：双站点架构去除了模式切换复杂度、L2 多题让数据模型更紧凑、9 项简化大幅减少代码量。

---

## 🟢 MVP：核心闭环（5-6 周）

### 目标

> **「能让孩子玩通一个 L1 探险」**——验证产品方向是否对路，验证孩子是否真喜欢这个玩法。

### 范围

#### 编辑端（admin/，约 2.5-3 周）

| 功能 | 估时 | 复杂度 |
|------|------|--------|
| 项目结构 + 路由 + 基础导航 | 2-3 天 | ⭐⭐ |
| L1 探险 CRUD | 2-3 天 | ⭐⭐ |
| L2 点位 CRUD + 上移/下移 + 多题绑定 | 4-5 天 | ⭐⭐⭐ |
| L3 题库 CRUD + usedCount 统计 + 筛选 | 3-4 天 | ⭐⭐⭐ |
| CSV 批量导入题目（PapaParse） | 2-3 天 | ⭐⭐ |
| 6 位数字码生成（含冲突检查） | 1 天 | ⭐ |
| 单张二维码生成（PNG） | 1-2 天 | ⭐⭐ |
| 浏览器打印（window.print + CSS） | 2-3 天 | ⭐⭐ |
| JSON 导出（全部 + 单 L1 + 仅题库） | 2-3 天 | ⭐⭐ |
| JSON 导入（完整覆盖模式 + 二次确认） | 2 天 | ⭐⭐ |
| 删除前确认 + 5 秒 Undo toast | 1-2 天 | ⭐⭐ |
| 桌面端响应式 + 简单移动端适配 | 2-3 天 | ⭐⭐ |

**编辑端小计：约 2.5-3 周**

#### 游戏端（play/，约 2-2.5 周）

| 功能 | 估时 | 复杂度 |
|------|------|--------|
| PWA 框架（manifest、Service Worker） | 1-2 天 | ⭐⭐ |
| 启动页 + 探险选择页 | 1-2 天 | ⭐ |
| 开始游戏 + 提示展示页 | 1 天 | ⭐ |
| 扫码功能（html5-qrcode） | 3-4 天 | ⭐⭐⭐ |
| 6 位数字码输入 fallback | 1-2 天 | ⭐⭐ |
| 「找不到？」按钮 + L1 密码跳过 | 1 天 | ⭐ |
| 顺序验证 + 数据库查询识别 | 1-2 天 | ⭐⭐ |
| L2 内多题连答主流程 | 3-4 天 | ⭐⭐⭐ |
| 单选题答题 + 3 次容错 + 保底 | 2-3 天 | ⭐⭐ |
| 求助机制（3 次，标记 usedHelp） | 1-2 天 | ⭐⭐ |
| 积分计算（含 usedHelp 规则） | 1-2 天 | ⭐⭐ |
| L2 通关拍合影（可选不阻塞） | 2-3 天 | ⭐⭐⭐ |
| 极简通关页 | 1-2 天 | ⭐⭐ |
| 暂停 / 续玩（自动保存） | 2 天 | ⭐⭐ |
| 应急通关（裁判模式） | 1-2 天 | ⭐⭐ |
| 导入 JSON 数据（含未完成游戏保护） | 1-2 天 | ⭐⭐ |
| Eruda 调试集成 | 0.5 天 | ⭐ |

**游戏端小计：约 2-2.5 周**

#### 共享层（shared/，约 1 周，散布在各功能中）

| 内容 | 估时 |
|------|------|
| 数据模型定义（Level、Point、Question 等） | 1-2 天 |
| 短 ID 生成工具 | 0.5 天 |
| IndexedDB 封装（Dexie.js） | 2 天 |
| JSON 导入导出工具 | 2 天 |
| 第三方库本地化 | 0.5 天 |

#### 真实测试 + 修 bug

约 0.5-1 周。至少 2 次真实户外测试 + 修 bug + 体验调整。

### MVP 总时长

> **5-6 周**（基于每周 5-8 小时投入）

**关键里程碑**：

| 周 | 里程碑 |
|----|--------|
| 第 1 周末 | 双站点架构搭好，编辑端能建 L1 + L2 |
| 第 2 周末 | 编辑端能绑定题目 + 生成二维码 + JSON 导出 |
| 第 3 周末 | 游戏端 PWA 跑起来，能扫码进入 L2 |
| 第 4 周末 | 完整跑通一个 L1（多 L2 多题） |
| 第 5-6 周 | 真实测试 + 修 bug + 打磨 |

### MVP 验证标准

- [ ] 家长在电脑端 15 分钟内完成 1 个完整 L1 的搭建（3 个 L2、共 9 道题）
- [ ] 二维码打印清晰 + 6 位短码可读
- [ ] JSON 成功从电脑传到 iPad
- [ ] 孩子能独立完成一个 L1（除了求助 / 扫不出码时找家长）
- [ ] 通关后看到总分 + 用时 + L2 合影集
- [ ] 全程不需要网络

## 🚀 V1：多人 + 完整体验（3-4 周）

### 触发条件

> MVP 通过测试，孩子玩了 5 次以上仍有兴趣。

### 范围

| 功能 | 估时 |
|------|------|
| 多屏通关流程 | 3-4 天 |
| 答题之星揭晓动画 | 1-2 天 |

#### 其他增强（约 1 周）

| 功能 | 估时 |
|------|------|
| 题库标签随机抽题 | 1-2 天 |
| 题目插图导入 | 2 天 |
| 完整 PDF 批量打印 | 2 天 |
| JSON 智能合并（覆盖/跳过/副本） | 2 天 |
| 朗读功能 + 降级 | 1-2 天 |

| 编辑端"最近删除" 7 天回收站 | 1-2 天 |

| 二维码内容保护（未进入会话提示） | 1 天 |
| 头像贴纸滤镜 | 2 天 |
| L1 导出含图片（zip 包） | 2 天 |

### V1 总时长

> **3-4 周**

**关键里程碑**：

| 周 | 里程碑 |
|----|--------|
| 第 1 周末 | 多人模式开局 + 答题归属跑通 |
| 第 2 周末 | 投票 + 颁奖 + 保底奖完整 |
| 第 3-4 周 | 其他增强 + 测试 |

### V1 验证标准

- [ ] 3-4 个孩子一起玩，全员能参与答题
- [ ] 通关后所有孩子都有奖
- [ ] 投票环节孩子能正常完成
- [ ] 题库 100+ 道题，能按标签随机出
- [ ] 至少部分 iPad 听到了朗读

## 📈 时间预估说明

### 让时间变长的因素

- 第一次开发学习曲线（前 2 周尤其慢）
- iPad 真机调试反馈循环长
- iOS Safari 兼容性 bug
- 工作 / 家庭事务
- 完美主义陷阱

### 让时间变短的因素

- Claude Code AI 辅助
- 纯前端 PWA 技术栈成熟
- **双站点架构去除复杂度**（v1.3 新增）
- **9 项简化大幅减少代码量**（v1.3 新增）
- 自用 App 没有兼容性压力
- 不需要上架审核

### 复杂度等级

- ⭐ 简单（1 天以内）
- ⭐⭐ 中等（2-3 天）
- ⭐⭐⭐ 较复杂（3-5 天）
- ⭐⭐⭐⭐ 复杂（1-2 周）

## 🎯 不同时间线建议

### 「两个月看到 MVP」

严格按 MVP 范围，新想法进 V1 等待区。

### 「细致打磨」

MVP 内增加 1 周打磨缓冲，每个功能完成后真机测试。

### 「时间不稳定」

每个 commit 都保持「可工作」状态。每次回归开发先 30 分钟读 PROGRESS.md。

## 🚦 何时停下评估

- MVP 做到第 10 周还没跑通核心闭环
- 真实测试 2-3 次后孩子明显不感兴趣
- 自己已经 1 个月没动开发
- 出现严重技术死胡同
