# 功能名：V1-24 填空题（L3 新题型）

> 编辑端新建填空题（type='fill'，fillAnswers[] 可接受答案数组），游戏端文本框输入作答。
> 答案匹配去前后空格、不区分大小写。不升 schema（fillAnswers 附加字段）。

---

## 编辑端测试（桌面 Chrome + iPad Safari）

### 新建填空题
- [ ] 题型区点「填空题」→ 选项区和正确答案区消失，出现「填空答案」区
- [ ] 填写题目 + 学科 + 至少 1 个可接受答案 → 点保存 → 返回列表，该题显示为填空题
- [ ] 重新编辑该题 → 答案区显示之前填的内容（没丢）
- [ ] 「＋ 添加可接受答案」加到第 2、3 个答案 → 保存 → 重开仍在
- [ ] 删除多余答案（点 ✕）→ 只剩 1 个时 ✕ 不出现（不能删到 0 个）
- [ ] 不填答案就保存 → 弹「请至少填写一个可接受答案」
- [ ] 切回「选择题」→ 选项区和正确答案区恢复，填空答案区消失
- [ ] 切回「判断题」→ 同上，填空答案区消失
- [ ] 编辑旧选择题 → 打开正常，没出现填空答案区
- [ ] F12 Console 无红色错误

## 游戏端测试（按 §5.7 三段结构）

### 桌面浏览器测试（Windows + Chrome，Live Server）

#### 基础答题流程
- [ ] 含填空题的探险导入游戏端，进入答题页
- [ ] 填空题站点：看到文本输入框 + 「✓ 提交答案」按钮，**没有** A/B/C/D 选项按钮
- [ ] 输入正确答案（完全匹配）→ 答对，进下一题或拍照页
- [ ] 输入错误答案 → input 抖动，显示「答案不正确」提示；0.6s 后 input 清空可重输
- [ ] 连输 3 次错误 → 保底通过（0 分），显示「没关系，下次记住啦！」+ 正确答案 + 解析（有则显示）+ 「好的，记住了！」按钮
- [ ] 点「好的，记住了！」→ 继续下一题

#### 答案匹配变体
- [ ] 正确答案「北京」，输入「北京」→ 答对（精确匹配）
- [ ] 输入「  北京  」（前后空格）→ 答对（去空格后匹配）
- [ ] 输入「北京市」（如果 fillAnswers 有第 2 个答案「北京市」）→ 答对
- [ ] 英文答案「apple」，输入「Apple」「APPLE」→ 答对（大小写不影响）
- [ ] Enter 键触发提交（不用点按钮）→ 正常答题

#### 求助按钮
- [ ] 填空题答错后点「求助」→ 弹确认卡片
- [ ] 确认求助 → 2s 后恢复，fill input 可用（没变成 disabled）
- [ ] 点「不用了」→ input 立即恢复可用
- [ ] 求助后答对 → 得分 3（usedHelpThisQuestion=true）

#### 计分 + 成绩单（V1-02 联动）
- [ ] 一次答对 → 10 分；错 1 次后答对 → 5 分；用求助 → 3 分；保底 → 0 分
- [ ] 通关后成绩单（V1-02）：填空错题的 wrongChoices 显示输入的文字（不是 A/B/C/D）
- [ ] 成绩单二级展开：正确答案显示 fillAnswers[0]；解析有则显示

#### 旧题型不受影响
- [ ] 同一探险含选择题 → 选项按钮正常显示，答题流程无变化
- [ ] 含判断题 → 同上

- [ ] F12 Console 无红色错误
