### 1.5 把 emoji 换成图片（PNG / SVG）

可以。但要分清两件事，否则白担心存储：

#### 这些图标换成图片，**不会**让导出的 JSON 文件变大

按钮、页面、探险列表的图标，是**全站共用的固定素材**，放在仓库的 `play/assets/` 里，**不会**被存进每个探险的数据（IndexedDB / 导出 JSON）。所以换成图片，**不影响你 AirDrop 传的 JSON 文件大小、不占探险数据空间**。真正占空间的是「探险地图底图」（那个是 base64 存进 JSON 的，编辑端已自动压到 1600px）。

#### 操作步骤（以「拍照按钮换成图片」为例）

1. 把图片放进 `play/assets/icons/`，例如 `play/assets/icons/camera.svg`
2. 在 HTML 里把 emoji 换成 `<img>`，例如 `photo.html` 第 41 行：
   ```html
   <!-- 原来 -->
   <button class="btn-primary btn-success" id="btn-take-photo">📸 拍照</button>
   <!-- 改成 -->
   <button class="btn-primary btn-success" id="btn-take-photo">
     <img src="../assets/icons/camera.svg" alt="" style="width:24px;height:24px;vertical-align:middle"> 拍照
   </button>
   ```
3. **离线关键一步**：把新图片加进 `play/service-worker.js` 的 `PRECACHE_URLS` 列表（第 13 行起），并把第 9 行的 `CACHE_VERSION` 版本号 +0.01，否则 iPad 飞行模式下图片不显示。例如：
   ```js
   './assets/icons/camera.svg',
   ```

> 探险列表那种「数组选图标」的，换图片要把 `ICONS` 从「emoji 字符数组」改成「图片路径数组」，并把渲染处从 `textContent = emoji` 改成插入 `<img>`。这个改动稍大，建议要做时单独提，我帮你改成统一逻辑。

#### 图片尺寸 / 格式建议（省空间又清晰）

| 格式 | 推荐度 | 说明 |
|------|--------|------|
| **SVG** | ⭐ 首选 | 矢量，任意放大都清晰，单个通常 1–5 KB，最省空间 |
| **PNG** | 可用 | 导出成「显示尺寸的 2 倍」就够。比如按钮图标显示 24px → 做 48×48；探险大图标显示 72px → 做 144×144。用 [tinypng.com](https://tinypng.com) 压一下，单个约 5–20 KB |
| **大照片当图标** | ❌ | 千万别拿手机原图（几 MB）当图标，又糊又占地方 |

整套 UI 图标加起来通常不到 100 KB，相对地图底图可以忽略。

### 3.3 通用：怎么自己改任何一段文字的大小 / 间距 / 字体

1. **先确定改的是哪段文字**：在页面上看到的文字，去对应的 `.html` 文件搜中文，找到它外面的 `class="xxx"`。
2. **去 `play/styles/main.css` 搜 `.xxx`**，看到这几个属性就能调：
   - `font-size: 20px;` → **字号**，数字越大越大
   - `line-height: 1.8;` → **行高**（一行多高），数字越大越松
   - `margin: ...; padding: ...;` → **间距**（margin 是外边距、padding 是内边距），数字越小越紧凑
   - `color: #2C3E50;` → **颜色**
   - `text-align: center;` → **对齐**
   - `font-weight: 700;` → **粗细**（400 常规、700 粗、900 最粗）
3. **改字体**：加一行 `font-family`。本项目约定用**系统字体**（不引字体文件，省空间）：
   ```css
   font-family: -apple-system, "PingFang SC", "Microsoft YaHei", sans-serif;
   ```
   （iPad 上是苹方，Windows 上是微软雅黑。想要别的字体得引字体文件，会增加加载和缓存体积，本项目暂不建议。）

> 有些文字的样式是**直接写在 HTML 的 `style="..."` 里**的（叫内联样式），不在 main.css。比如 photo.html 第 30 行的「通关文案」就是 `style="...font-size:17px..."`。这种直接在那一行的 `style` 里改数字即可。

## 4. 通用：怎么调整「任何」UI 元素的大小（万能方法）

不管是按钮、图标、图片、卡片、头像还是间距——调大小都是**同一套路**：找到它的 CSS 类 → 改 `width / height / font-size / padding`。

### 4.1 三步定位法

1. **在页面上看到那个元素**，记住它附近的文字。
2. 用 VS Code 打开对应页面 `.html`（启动 `index.html`、列表 `select-level.html`、开始 `start-level.html`、找站 `hint.html`、扫码 `scan.html`、答题 `quiz.html`、拍照 `photo.html`、通关 `victory.html`），`Ctrl+F` 搜那段中文，看它外层的 `class="xxx"`。
3. 去 `play/styles/main.css` 搜 `.xxx`，改这些数字（单位默认 px，除非写了 `vh`/`%`）：
   - `width` / `height` → 元素宽高
   - `font-size` → 文字 / emoji 大小
   - `padding` → 内部留白（撑大元素 / 拉开内部间距）
   - `margin` → 外部间距（与别的元素的距离）
   - `border-radius` → 圆角大小
   - `max-height` / `max-width` → 图片等的尺寸上限（如提示图 `55vh`）

> **例外（内联样式）**：有些元素样式**直接写在 HTML 的 `style="..."` 里**，不在 main.css。判断方法：搜到那段 HTML，若它自己带 `style="...width:..."` 就**改这一行**；若只有 `class="xxx"` 没有 `style`，就去 main.css 改 `.xxx`。
> 例：探险列表的**封面缩略图**尺寸写在 `select-level.html` 的 `<img ... style="width:60px;height:60px...">` 里（内联）；而 **emoji 图标**尺寸在 main.css 的 `.level-icon`。

| 探险列表 L1 封面缩略图 | select-level.html `<img>` 内联 | `width/height:60px` |

## 附录：改完怎么生效（重要！）

游戏端是 PWA，有缓存。**只改文件不升版本号，iPad 上可能还是旧样子。** 改完按这个顺序：

1. **改了 `.js` 或 `.css`**：升 `play/service-worker.js` 第 9 行的 `CACHE_VERSION`（小数 +0.01，例如 `v2.21` → `v2.22`）。
2. **新增了图片/文件**：把路径加进同文件的 `PRECACHE_URLS` 列表。
3. 推送前：改 `shared/version.js` 的 `BUILD_TAG`（项目约定，方便在设置页确认 iPad 跑的是不是新版）。
4. **推 GitHub**（详细步骤见下方「附录二」）→ iPad 上**有网**打开一次（让 Service Worker 装新缓存）→ 之后飞行模式也是新样子。
5. 桌面调试时改 CSS/HTML 看不到变化，按 `Ctrl+Shift+R` 强制刷新。

> 只改纯 HTML 文字（比如换个 emoji 字符）也建议升一下 `CACHE_VERSION`，确保 iPad 拿到新页面。

## 附录二：怎么推 GitHub（详细版）

推 GitHub = 把你电脑上改好的文件「上传」到云端仓库。GitHub Pages 会自动用最新文件更新网站，
通常 **1–2 分钟**后，iPad 联网打开就是新版了。下面三种方式任选，**推荐方式 A**。

### 方式 A：直接让我（Claude Code）帮你推 ⭐ 最省事

改完后，你只要跟我说一句：

> 「帮我推到 GitHub，commit 写 xxx」

我就会自动执行下面方式 C 的全部命令并确认结果。你什么都不用记。
（项目约定：commit 用单行英文，我会照办。）

### 方式 B：用 VS Code 左侧「源代码管理」按钮（图形界面，不打字）

1. 点 VS Code 最左侧竖排图标里的**「源代码管理」**（一个像树杈的图标，或按 `Ctrl+Shift+G`）。
2. 上方会列出所有改动的文件。在输入框里**填一句话**说明这次改了啥（中英文都行，本项目约定英文）。
3. 点输入框上方的 **✓ 提交（Commit）**。若提示「没有暂存的更改，是否暂存全部」，选**「是」**。
4. 再点 **「同步更改」/「Sync Changes」**（带上下箭头的按钮），把本地提交推到 GitHub。
5. 等 1–2 分钟，GitHub Pages 自动更新。

### 方式 C：用 PowerShell 敲命令（三行搞定）

在 VS Code 里按 `` Ctrl+` `` 打开终端，确认左边路径是 `D:\Vibecoding\treasure-hunt`，依次粘贴：

```powershell
# 1) 把所有改动加入待提交（. 表示「全部改动的文件」）
git add .

# 2) 提交并写一句说明（引号里换成你这次改了啥，单行英文）
git commit -m "tweak: adjust map size and victory/photo text styles"

# 3) 推到 GitHub
git push
```

**怎么算成功**：`git push` 之后看到类似 `main -> main` 的字样就成了。
之后打开 `https://tlook-spec.github.io/treasure-hunt/play/`，**等 1–2 分钟**再刷新（手机端关掉 App 重开）。

### 常见情况

| 现象 | 原因 / 怎么办 |
|------|--------------|
| `git push` 让我输用户名密码 | 第一次需要登录授权。可在终端输入 `! gh auth login` 跟引导走，或直接让我帮你推 |
| iPad 打开还是旧样子 | ① 没等够 1–2 分钟　② 没升 `CACHE_VERSION`　③ PWA 没彻底关掉重开（从后台划掉再点图标）④ 飞行模式下第一次拿不到新版，要先联网开一次 |
| 桌面 Chrome 还是旧样子 | 按 `Ctrl+Shift+R` 强刷；或 F12 → Application → Service Workers → 勾「Update on reload」 |
| 不小心改坏了想撤销 | 还没 commit：`git checkout -- 文件名` 还原单个文件；拿不准就问我，别乱用 `reset --hard` |

> 小结：**最省心就是改完跟我说「帮我推 GitHub」**，其余交给我。
