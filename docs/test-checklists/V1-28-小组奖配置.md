# 功能名：V1-28 编辑端 L1「通关小组奖（可选）」配置

> 在 L1 新建/编辑表单（`admin/pages/levels.html` + `admin/scripts/level-form.js`）「探险地图」分组下方
> 新增「🏆 通关小组奖（可选）」分组：奖名 `groupAwardName`（单行文本）+ 奖图 `groupAwardImage`（压 ≤800 base64）。
> 两个新字段，**不动 schema、不动导出逻辑**（随 exportFull / importSingleLevel 自然进出）。游戏端 V1-29 没配走默认。

---

## 桌面浏览器测试（Windows + Chrome）

### 填奖名 + 传奖图 → 保存 → 重开值还在
- [ ] 编辑某 L1 → 看到「🏆 通关小组奖（可选）」分组在「探险地图」下方
- [ ] 填奖名「丛林勇士勋章」+ 选一张奖图 → 出现缩略图预览（110×110）+「🗑️ 移除奖图」按钮
- [ ] 点保存 → 重新打开该 L1 编辑 → 奖名还在、奖图预览还在
- [ ] F12 → Application → IndexedDB → treasure-hunt-admin-db → levels → 该条：
  - [ ] `groupAwardName` = 填的奖名
  - [ ] `groupAwardImage` 是 base64 字符串，且压缩后长边 ≤800（可在 console 用 `new Image().src=...` 验尺寸）

### 「移除」奖图 → 变 null
- [ ] 编辑有奖图的 L1 → 点「🗑️ 移除奖图」→ 预览消失
- [ ] 保存 → F12 看 `groupAwardImage` = `null`

### 只填奖名不传图 / 都留空
- [ ] 只填奖名、不传图 → 保存不报错；`groupAwardName` 有值、`groupAwardImage` = null
- [ ] 奖名也留空 → 保存不报错；`groupAwardName` = `""`、`groupAwardImage` = null

### 老 L1（没这俩字段）兼容
- [ ] 打开一个本功能之前建的 L1 → 不报错，奖名框为空、无奖图预览
- [ ] 不动奖项直接保存 → 不报错（之后该 L1 会带上 `groupAwardName:""` / `groupAwardImage:null`）

### 随导出验证（不改导出逻辑）
- [ ] 给某 L1 配好奖名 + 奖图 → 在 L1 详情页「📤 导出」单 L1 JSON
- [ ] 打开导出的 .json 文件 → `data.levels[0]` 里能看到 `groupAwardName` 和 `groupAwardImage`（base64）

### 新建 L1
- [ ] 新建一个 L1，配奖名 + 奖图 → 保存 → 列表/详情正常，编辑重开值在
- [ ] 新建 L1 不配奖 → 保存不报错
