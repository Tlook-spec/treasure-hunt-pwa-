# 功能名：play 设置页「🧹 清理游玩垃圾」（清孤儿头像 + 孤儿合影）

> 设置页「存储维护」区新增按钮：扫所有 `sessions`，删掉**没被任何存档引用**的照片（孤儿头像 / 孤儿合影）。
> 探险、当前未完成存档、还被引用的合影、历史存档记录**都不动**。当前数据卡新增「📷 张照片」统计。

---

## 桌面浏览器测试（Windows + Chrome）

### 制造孤儿 → 清理
- [ ] 多人玩一局拍几张合影 + 设头像 → 设置页「📷 张照片」数 > 0
- [ ] 整库覆盖导入一份 JSON（清掉 sessions）→ 此时照片变孤儿但数字不变（导入不删照片）
- [ ] 点「🧹 清理游玩垃圾」→ 确认 → 弹「🧹 清理了 N 张照片，腾出约 X MB」
- [ ] 「📷 张照片」数字同步减少（应归 0 或只剩仍被引用的）
- [ ] F12 → IndexedDB → photos 表：孤儿记录已删

### 不误删「在用」的照片（关键）
- [ ] 玩到一半（有合影、未通关，存档 in_progress）→ 点清理
- [ ] 当前存档的合影**仍在**（被 pointRecords.completionPhotoId 引用 → 不算孤儿）
- [ ] 续玩该存档正常，合影还能看到
- [ ] 打通一局（completed 存档）→ 点清理 → 该局合影**仍在**（completed 存档保留，照片被引用）

### 没垃圾时
- [ ] 全新导入后没玩过 / 已清理过 → 点清理 → 弹「✨ 没有可清理的垃圾，已经很干净啦！」，不报错

### 不破坏其他
- [ ] 清理后探险数（📝/❓）不变，能正常开局玩
- [ ] 拼音开关、导入、追加导入都正常
- [ ] F12 Console 无红色错误
