# 寻宝游戏 PWA - 版本路线图

> 本文档对应 PRD v1.7 的版本规划,详细列出 MVP、V1、V2 的功能清单 + 预估开发时间。
> 时间预估基于「零基础 + Claude Code 辅助 + 每天 1-2 小时 / 每周 5-8 小时」的投入强度。
>
> **当前版本:v1.7**(基于 PRD v1.7 重新估时)

---

## 📊 总览

```
现在 ──┬── MVP (5-6 周) ──┬── V1 (4-5 周) ──┬── V2 (4-6 周) ──→ 长期维护
       │                  │                 │
       │ 单人能玩          │ 多人 + 完整庆典  │ 应急通关裁判模式
       │ 双站点架构        │ + 探险地图       │ + GitHub 同步
       │ L2 多题 + 合影    │ + 单 L1 追加导入 │ + 海报 + 雷达
       │ CDN + 动态 SW    │                 │
```

| 版本 | 估时 | 里程碑 |
|------|------|--------|
| **阶段 0+1** | 完成中 | 环境就绪 + 双站点架构 + 设计 |
| **MVP** | 5-6 周 | 单人跑通一个 L1(含多 L2 + 多题) |
| **V1** | 4-5 周 | 多人 + 完整庆典 + 探险地图 |
| **V2** | 4-6 周 | 应急通关 + GitHub 同步 + 长期玩家档案 |
| **总计** | **13-17 周(3.5-4.5 个月)** | 完整可用 App |

> 💡 真实工期一般是 1.3-1.5 倍。**心理预期 5-7 个月**。
>
> 🆚 比 v1.1 路线图缩短 1-2 周。原因:
> - MVP CDN + 动态缓存 SW(省去本地化库 + 手写缓存列表的时间)
> - MVP 只做整库覆盖导入(单 L1 追加推 V1)
> - 应急通关裁判模式从 V1 推到 V2
> - 大量功能彻底删除(朗读、zip 包、JSON 智能合并等 11 项)

---

## 🟢 MVP:核心闭环(5-6 周)

### 目标

> **「能让孩子玩通一个 L1 探险」**——验证产品方向是否对路,验证孩子是否真喜欢这个玩法。

### 范围

#### 编辑端(admin/,约 2-2.5 周)

| 功能 | 估时 | 复杂度 |
|------|------|--------|
| 项目结构 + 路由 + 基础导航 | 2-3 天 | ⭐⭐ |
| L1 探险 CRUD(隐藏地图字段) | 2-3 天 | ⭐⭐ |
| L2 点位 CRUD + 上移/下移 + 多题绑定 | 4-5 天 | ⭐⭐⭐ |
| L3 题库 CRUD + usedCount 统计 + 按 usedCount 筛选 | 2-3 天 | ⭐⭐ |
| CSV 批量导入题目(UTF-8 提示文案 + 教学截图) | 2-3 天 | ⭐⭐ |
| 6 位数字码生成(含冲突检查) | 1 天 | ⭐ |
| 单张二维码生成(PNG) | 1-2 天 | ⭐⭐ |
| 浏览器打印(window.print + CSS):二维码 | 2 天 | ⭐⭐ |
| 应急小抄打印(单 L1 数字码清单) | 1 天 | ⭐ |
| JSON 整库导出 | 1-2 天 | ⭐⭐ |

#### 真实测试 + 修 bug

约 0.5-1 周。至少 2 次真实户外测试 + 修 bug + 体验调整。

### MVP 总时长

> **5-6 周**(基于每周 5-8 小时投入)

**关键里程碑**:

| 周 | 里程碑 |
|----|--------|
| 第 1 周末 | 共享层完成,编辑端能建 L1 + L2 |
| 第 2 周末 | 编辑端能绑定题目 + CSV 导入 + 二维码 + JSON 导出 |
| 第 3 周末 | 游戏端 PWA 跑起来,能扫码进入 L2 |
| 第 4 周末 | 完整跑通一个 L1(多 L2 多题 + 拍合影 + 通关页) |
| 第 5-6 周 | 真实测试 + 修 bug + 打磨 |

### MVP 验证标准

- [ ] 家长在电脑端 15 分钟内完成 1 个完整 L1 的搭建(3 个 L2、共 9 道题)
- [ ] 二维码打印清晰 + 6 位短码可读
- [ ] 应急小抄能打印 + 含所有 L2 数字码
- [ ] JSON 成功从电脑传到 iPad
- [ ] 孩子能独立完成一个 L1(除了求助 / 扫不出码时找家长)
- [ ] 通关后看到总分 + 用时 + L2 合影集
- [ ] 全程不需要网络(首次访问后)

## ❌ 已确认永远不做(v1.7 删除清单)

下列功能在 v1.7 已从所有版本中彻底移除,不放在任何版本里:

| 功能 | 删除理由 |
|------|---------|
| 朗读功能(Web Speech API) | 4-12 岁孩子基本认字,iOS 中文 TTS 质量一般 |
| 完整 PDF 批量打印 | MVP 的 window.print() 足够用 |
| JSON 智能合并(覆盖/跳过/副本模式) | V1 用「按 ID 判重 + 二次确认」即可 |
| 退出玩家重新加入 | 真实场景概率极低 |
| 头像贴纸滤镜 | 锦上添花,非核心 |
| 编辑端「最近删除」7 天回收站 | JSON 备份已经够兜底 |
| 编辑端可选密码保护 | 自用场景意义不大 |
| 拖拽排序(L2、题目) | 上移/下移按钮足够 |
| L1 导出 zip 包 | 统一 base64 进 JSON,避免双管线 |
| CSV 编码自动检测(jschardet) | 文案提示用户保存 UTF-8 即可 |
| 二维码内容保护(未进入会话提示) | 自用场景无意义 |

## 📈 时间预估说明

### 让时间变长的因素

- 第一次开发学习曲线(前 2 周尤其慢)
- iPad 真机调试反馈循环长
- iOS Safari 兼容性 bug
- 工作 / 家庭事务
- 完美主义陷阱

### 让时间变短的因素(v1.7 新增)

- Claude Code AI 辅助
- 纯前端 PWA 技术栈成熟
- 双站点架构去除复杂度
- **CDN 引入 + 动态缓存 SW(无需手写本地化和缓存列表)**
- **MVP 只做整库覆盖导入,逻辑简单**
- **11 项彻底删除功能,大幅减少代码量**
- 自用 App 没有兼容性压力
- 不需要上架审核

### 复杂度等级

- ⭐ 简单(1 天以内)
- ⭐⭐ 中等(2-3 天)
- ⭐⭐⭐ 较复杂(3-5 天)
- ⭐⭐⭐⭐ 复杂(1-2 周)

## 🎯 不同时间线建议

### 「两个月看到 MVP」

严格按 MVP 范围,新想法进 V1 等待区。

### 「细致打磨」

MVP 内增加 1 周打磨缓冲,每个功能完成后真机测试 + 写测试 checklist。

### 「时间不稳定」

每个 commit 都保持「可工作」状态。每次回归开发先 30 分钟读 PROGRESS.md。
