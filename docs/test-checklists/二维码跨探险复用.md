# 功能名：二维码数字码手动填写 + 跨探险复用

> 背景：家长反馈想让打印好的二维码环保复用（同一批纸质二维码换个探险接着用）。
> 方案：点位表单数字码从「全库自动生成、不可改」改为「可编辑，默认自动生成」，
> 唯一性检查从「全库」收窄到「同一探险内」；游戏端扫码/输码改为「先定位本探险点位，
> 再在其中按码匹配」，避免不同探险复用同码后互相认错。

## 前置条件

- 编辑端已有至少 2 个探险（L1），用于测试跨探险复用同一个码

---

## 编辑端（admin/pages/levels.html + point-form.js）

### 桌面 Chrome 测试

- [ ] 新建点位 → 「二维码数字码」输入框**自动预填**一个 6 位数字（非空）
- [ ] 点「🔀 随机生成」→ 码变成另一个 6 位数字
- [ ] 手动在输入框粘贴非数字字符（如字母/符号）→ 自动被过滤，只保留数字
- [ ] 手动输入超过 6 位数字 → 自动截断为 6 位
- [ ] 保存点位 → 弹窗中该码正确显示，IndexedDB 中 code 字段与输入一致（含前导 0，如 "000123"）
- [ ] 同一探险内新建第二个点位，手动填入**已被第一个点位占用**的码 → 保存时弹出提示「已经被本探险内其他点位使用了」，保存失败
- [ ] 换一个没被占用的码 → 保存成功
- [ ] 编辑已有点位，把 code 改成本探险内其他点位的码 → 同样被拒绝
- [ ] **在另一个探险**里手动填入第一个探险已用过的码 → 保存成功（跨探险允许复用）
- [ ] 编辑已有点位 → code 输入框正确显示原有的码
- [ ] F12 Console 无红色错误
