## 前置条件
已导入含 ≥2 个点位的探险 JSON。

## 桌面浏览器测试（Windows + Chrome）

### 无未完成存档时
- [ ] 选择探险页顶部**没有**蓝色横幅
- [ ] 探险卡片列表正常显示
- [ ] F12 Console 无红色错误

### 有未完成存档时（模拟方式：开始一个探险但不通关，再回到选择页）
- [ ] 选择探险页顶部出现蓝色续玩横幅
- [ ] 横幅标题：「⏯ 继续上次的『探险名』？」，探险名正确
- [ ] 横幅副标题：「已到第 N 站」，N = 当前站号（1起算）
- [ ] 有两个按钮：「继续」和「放弃」

### 「继续」按钮逻辑（在不同中断点测试）

**中断点 A：已创建 session 但尚未扫码进第1站（hint.html → 返回 → 选择页）**
- [ ] 点「继续」→ 跳转到 `hint.html?sessionId=xxx`（提示页）

**中断点 B：已扫码进第1站、题目答到一半（quiz.html → 返回 → 选择页）**
- [ ] 点「继续」→ 跳转到 `quiz.html?sessionId=xxx`（答题页）
- [ ] 进入后看到「继续上次题目」的当前题（currentQuestionIndex 正确）

**中断点 C：第1站所有题答完、拍照/跳过前（拍照页返回 → 选择页，或直接改 session 模拟）**
- [ ] 点「继续」→ 跳转到 `photo.html?sessionId=xxx`（拍照页）

### 「放弃」按钮
- [ ] 点「放弃」→ 弹出浏览器 confirm 对话框，文字包含「确定放弃」
- [ ] 点「取消」→ 对话框消失，横幅保持不变
- [ ] 点「确定」→ 横幅消失，F12 IndexedDB 里 `sessions` 表该记录被删除
- [ ] 放弃后刷新页面 → 横幅不再出现

### 多个未完成存档（同时开了两个探险存档）
- [ ] 只显示 `startedAt` 最新的那条存档横幅
- [ ] 横幅内探险名与最新存档匹配

### M18 联动：导入保护
- [ ] 设置页导入 JSON 时若有 in_progress session → 弹出二次确认（M18 已有逻辑，这里验证未受 M26 影响）

## 备注

- `currentPointIndex` 从 0 起算，横幅显示「已到第 N 站」时 N = `currentPointIndex + 1`
- 放弃后存档从 IndexedDB `sessions` 表彻底删除（不是标记 status=abandoned）
