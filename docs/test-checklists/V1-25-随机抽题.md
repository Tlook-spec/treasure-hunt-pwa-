# 功能名：V1-25 题库随机抽题

> 编辑端 L2「选题面板」加「🎲 随机抽 N 道」按钮：复用面板现有的学科/难度/题干搜索筛选当抽题范围，
> 优先抽 usedCount<2 的题（软避让，不够再补已用多的）。抽到的加入勾选，家长仍需点「确认添加」。
> **纯编辑端**：admin 无 Service Worker，不升 CACHE_VERSION。

---

## 编辑端测试（桌面 Chrome + iPad Safari）

> 准备：题库里有足够多的题（含不同学科/难度；最好有几道已被多个点位引用、usedCount≥2 的）。

### 基础抽题
- [ ] 进入某 L1 → 编辑/新建 L2 → 点「从题库添加题目」打开选题面板
- [ ] 面板里筛选条下方出现「随机抽 [3] 道 🎲 随机抽取」一行
- [ ] 不设筛选，数量 3，点「随机抽取」→ 勾选里多出 3 道题，底部计数「已选 3 道题」
- [ ] 提示文字显示「已随机选中 3 道（已加入勾选，可再手动调整）」
- [ ] 点「确认添加」→ 这 3 道进入点位的题目绑定列表
- [ ] 保存点位 → 重新编辑该点位 → 这 3 道还在

### 按筛选范围抽
- [ ] 筛选「数学 + ★简单」→ 随机抽 3 → 抽到的都是数学简单题（符合筛选）
- [ ] 切换筛选到「语文」→ 再抽 → 抽到的是语文题（范围随筛选变）
- [ ] 搜索关键字过滤后再抽 → 只在搜索结果里抽

### 边界：数量不足 / 高频题
- [ ] 筛选到只有 2 道符合的题，数量填 5，点抽取 → 只抽到 2 道 + 提示「只抽到 2 道（符合条件的题不够 5 道）」
- [ ] 候选全是已用 ≥2 次的题 → 仍能抽到 + 提示补「其中 X 道是已用较多的题」
- [ ] 数量框填 0 或留空或超过 10 → 自动 clamp 到 1~10，不报错

### 不重复 + 混用
- [ ] 连续点两次「随机抽取」→ 第二次抽到的不和第一次重复（已勾选的被排除）
- [ ] 先手动勾 2 道，再随机抽 3 → 计数变 5，确认添加后 5 道都进列表
- [ ] 候选池被抽空后再点抽取 → 提示「没有可抽的题（调整筛选，或先清空已选）」

### 不破坏现有逻辑
- [ ] 同探险其他点位已占用的题 → 随机抽【不会】抽到它（occupied 排除）
- [ ] 已绑定到本点位的题 → 不会重复抽到
- [ ] 关闭面板再重开 → 数量复位 3、提示清空、已选清零（不跨点位保留）
- [ ] 纯手动勾选 + 确认添加流程仍正常（没被随机抽题破坏）
