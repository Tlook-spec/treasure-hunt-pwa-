# 功能名：数字码输入弹窗（M21）

> 覆盖范围：scan.html 底部两个备选按钮 + 数字码验证弹窗（孩子语境 / 家长语境）

## 前置准备

- 游戏端已导入 JSON（含 ≥1 个探险、≥3 个点位）
- 手头准备：当前站的 6 位数字码（查编辑端或应急小抄）、本探险另一站的码、任意 6 位乱码
- 从 hint.html 进入 scan.html（不需要摄像头，直接测底部按钮）

---

## 桌面 Chrome 测试

### 「⌨️ 扫不出？可以输入数字码」按钮（孩子语境）

**基本弹窗**
- [ ] 点击按钮 → 弹出白色弹窗，蓝色标题栏「输入这一站的 6 位数字码」
- [ ] 弹窗无正文说明（孩子模式不显示）
- [ ] 显示 6 个空格子 + 自定义数字键盘（1-9, ⌫, 0）
- [ ] 「取消」「确定」两个按钮可见
- [ ] 键盘按钮尺寸明显大（≥60px 高）

**输入体验**
- [ ] 点数字键盘 1-9 / 0 → 数字出现在对应格子里
- [ ] 第 1 格有蓝色高亮边框（激活框跟随输入位置移动）
- [ ] 输入 3 位后点「⌫」→ 删除最后一位，激活框后退
- [ ] 输入满 6 位 → 约 300ms 后自动触发验证
- [ ] 验证中「确定」变「⏳」并禁用（防重复提交）

**验证：当前站正确码**
- [ ] 输入 6 位正确码 → 弹窗关闭 → 跳转到 challenge.html（M22 前会 404，URL 正确即可）
- [ ] F12 Console 无报错

**验证：本探险其他站的码**
- [ ] 输入另一站的码 → **弹窗保持打开**（不关闭、不回摄像头）
- [ ] 红色提示「码不对哦，再看看二维码下面的数字」出现
- [ ] 6 个格子清空，可重新输入
- [ ] 再次输入正确码 → 成功跳转

**验证：查无此码的乱码**
- [ ] 输入随机 6 位数（不存在的码）→ 同上，弹窗保持，红色提示
- [ ] 格子清空可重输

**取消**
- [ ] 弹窗里点「取消」→ 弹窗关闭 → 扫码页回到「点击开始扫码」状态
- [ ] 再点底部按钮 → 弹窗可重新打开（状态干净，无上次输入残留）

### 「❓ 找不到二维码？」按钮（家长语境）

**基本弹窗**
- [ ] 点击按钮 → 弹出弹窗，**橙色**标题栏「请家长输入」
- [ ] 弹窗显示说明文字：「二维码找不到了？请爸爸妈妈输入这一站的 6 位数字码」
- [ ] 小字：「在打印的应急小抄、或电脑端 L1 详情页可以查到」
- [ ] 同样的 6 格 + 键盘 + 取消/确定（确定按钮为橙色）

**验证行为**
- [ ] 输入正确码 → 进入 challenge.html（与孩子模式相同，无「跳过」逻辑）
- [ ] 输入错误码 → 弹窗保持，红色提示「家长再仔细看下数字」（文案与孩子模式不同）
- [ ] 格子清空可重输

---

### 两种弹窗共用逻辑验证
- [ ] 两种弹窗的「取消」都能正常关闭

- [ ] 两种弹窗切换打开时各自的颜色和文案互不干扰
