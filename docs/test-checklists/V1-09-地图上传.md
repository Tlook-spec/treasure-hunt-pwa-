# 功能名：V1-09 编辑端 L1 地图字段 UI

> 改动范围：`admin/pages/levels.html`（模态框新增地图分组）、`admin/scripts/level-form.js`（新建/编辑/保存地图字段 + compressMapImage）

---

## 桌面浏览器测试（Windows + Chrome）

### 新建探险 — 传图压缩

- [ ] 打开「新建探险」弹窗 → 底部有「🗺️ 探险地图（可选）」分组可见
- [ ] 点「📷 选择图片」→ 选一张宽度 ≥3000px 的手绘图
- [ ] 选图后弹窗内出现缩略图预览，且出现「🗑️ 移除地图」按钮
- [ ] 保存后，在 Chrome DevTools → Application → IndexedDB → treasure-hunt-admin-db → levels → 找到刚建的 Level
  - [ ] `mapImage` 字段为 `data:image/jpeg;base64,...`（非 null）
  - [ ] 把 base64 字符串贴到浏览器地址栏可显示图片，目视长边 ≤1600px
- [ ] F12 Console 无红色错误

### 新建探险 — 不传图

- [ ] 「📷 选择图片」不操作，直接保存 → `mapImage = null`，不报错
- [ ] 再次打开「编辑」该探险 → 地图区无缩略图，「移除」按钮不显示

### 编辑探险 — 字号和颜色

- [ ] 新建探险时选「大」字号，保存
- [ ] 再次编辑 → 字号单选显示「大」已选中（值被正确读回）
- [ ] 修改「名称颜色（发现时）」为 `#FF0000`，保存，再打开 → 颜色值被记住
- [ ] 修改「名称颜色（完成后）」为 `#00AA44`，保存，再打开 → 颜色值被记住

### 移除地图

- [ ] 传图后点「🗑️ 移除地图」→ 缩略图消失，「移除」按钮隐藏
- [ ] 保存后再打开 → `mapImage = null`，地图区净空
- [ ] F12 Console 无红色错误

### 兼容性（老探险）

- [ ] 选择一个在 V1-09 之前创建的老探险点「编辑」→ 弹窗正常打开，无 JS 报错
- [ ] 老探险没有 `mapFontSize` 字段 → 字号默认显示「中」；没有颜色字段 → 颜色显示为默认色
