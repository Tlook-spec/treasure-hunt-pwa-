# 功能名：M13 JSON 整库导入导出

## 前置条件

- 编辑端已有至少 2 个探险（各含点位）、5+ 道题目

---

## 整库导出

- [ ] 进入「导入 / 导出 JSON」页面，显示导出 + 导入两个区块
- [ ] 点「📥 立即导出」→ 浏览器下载 `.json` 文件
- [ ] 文件名格式正确：`treasure-hunt-backup-YYYY-MM-DD.json`
- [ ] 按钮导出中变为「导出中…」禁用，完成后恢复
- [ ] 导出后显示「✅ 已导出 xxx.json」提示
- [ ] 用文本编辑器打开 JSON 文件，确认结构：
  - 有 `schemaVersion: "1.0"`
  - 有 `exportedAt`（时间戳）
  - 有 `exportType: "full"`
  - 有 `data.levels`、`data.points`、`data.questions` 三个数组
- [ ] F12 Console 无红色错误

---

## 整库导入 — 正常流程

- [ ] 点「📂 选择 JSON 文件」→ 文件选择对话框出现
- [ ] 选中上一步导出的 JSON 文件 → 文件名显示在按钮旁
- [ ] 自动校验通过 → 显示「步骤 2 · 文件信息」卡片
- [ ] 文件信息卡片显示：导出时间（可读格式）、包含 N 个探险/点位/题目
- [ ] 点「✅ 确认导入」→ 弹出二次确认弹窗
- [ ] 弹窗显示：当前数据库条数 + 将写入条数
- [ ] 弹窗显示「⚠️ 此操作无法撤销」
- [ ] 点「取消」→ 弹窗关闭，数据未变
- [ ] 点「确定替换」→ 导入执行，弹窗关闭
- [ ] 显示「✅ 导入成功！已写入 N 个探险…」
- [ ] 文件输入重置为「未选择文件」
- [ ] 前往探险管理 → 数据已恢复

---

## 整库导入 — 清空后恢复

- [ ] 在浏览器 DevTools 手动清空 IndexedDB（或用 Console 删除所有记录）
- [ ] 回到导入页，选择之前的备份 JSON
- [ ] 确认导入 → 数据完整恢复（探险数 / 点位数 / 题目数与导出时一致）

---

## 整库导入 — 异常情况

- [ ] 选择一个 `.txt` 文件（把 .txt 扩展名的文本直接上传）→ 显示「这不是有效的 JSON 文件」错误
- [ ] 选择一个内容合法但缺少 `schemaVersion` 的 JSON → 显示「缺少 schemaVersion」错误
- [ ] 选择 `schemaVersion` 为 `"2.0"` 的 JSON → 显示「不兼容的 App 版本」错误
- [ ] 选择 `exportType` 为非 `"full"` 的 JSON → 显示「不支持的导出类型」错误
- [ ] 出现错误后，步骤 2 预览不显示
- [ ] 错误后可重新选择文件，不需要刷新页面
