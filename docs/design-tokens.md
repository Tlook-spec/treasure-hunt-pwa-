# 视觉设计规范（design-tokens）

> 本文档是寻宝游戏 PWA 的视觉令牌定稿，对应 PRD §5。
> 字段已确认，可直接进项目。开发时建议把"色值"段抄进各端 CSS 的 `:root`。
> **MVP 范围**：用纯 CSS + emoji + 简单动效实现；自定义插画 / 故事配图 V1 引入（见版本路线图）。

---

## 一、配色

### 主色 · 明亮蓝（按钮、导航、主操作）
| 名称 | 色值 | 用途 |
|------|------|------|
| 主色 | `#4A90E2` | 主按钮底色、导航高亮、选项选中态 |
| 主色深 | `#3A7BC8` | 主按钮 hover/按下、选中态描边 |
| 主色浅 | `#7BB0EE` | 卡片标题底色、未选选项的边框 |

### 辅助 / 强调 / 错误
| 名称 | 色值 | 用途 |
|------|------|------|
| 辅助 · 暖橙 | `#F5A623` | 提示卡片、求助按钮、家长输入弹窗 |
| 强调 · 鲜绿 | `#7ED321` | "成功 / 前进"：扫码按钮、答对反馈 |
| 错误 · 柔和粉橙 | `#FF9F89` | 出错 / 删除（柔和，不用刺眼红，对孩子友好）|

### 中性色
| 名称 | 色值 | 用途 |
|------|------|------|
| 背景色 | `#FFFFFF` | 编辑端页面底色 |
| 卡片背景 | `#F8F9FA` | 卡片、左导航底色 |
| 文字主色 | `#2C3E50` | 正文、标题 |
| 文字次色 | `#6C757D` | 元信息、次要说明 |
| 边框色 | `#E9ECEF` | 卡片 / 输入框 / 分隔线 |

### 游戏端专用背景（规范补充项）
| 名称 | 色值 | 用途 |
|------|------|------|
| 游戏端暖背景 | `#FFF9E6` | 游戏端答题页等页面底色（淡黄暖色，比纯白更有童趣，与线框图一致）|

> 说明：编辑端用纯白 `#FFFFFF`（简约专业）；游戏端用淡黄 `#FFF9E6`（温暖童趣）。这是两端唯一的背景差异，其余颜色共用。

### 颜色语义约定（避免撞色）
- **蓝 = 主操作 / 选中**：选项选中态填主色蓝 + 白字，未选白底 + 浅蓝边。
- **绿 = 成功 / 前进**：只在"答对""扫码前进"时出现，不用于普通按钮（否则与答对撞色）。
- **橙 = 辅助 / 提示 / 家长语境**：提示、求助、家长输入。
- **粉橙 = 出错 / 删除**：温和提示，不制造"被骂"感。

## 二、字号
| 名称 | 大小 | 用途 |
|------|------|------|
| 大标题 | 32px | 启动页 App 名、通关页"通关啦" |
| 标题 | 24px | 页面主标题、游戏端题干 |
| 副标题 | 18px | 卡片标题、选项文字 |
| 正文 | 16px | 普通正文、进度栏 |
| 提示 | 14px | 元信息、次要说明 |
| 按钮 | 18px | 按钮文字 |

> 游戏端户外强光场景，题干、选项务必用大字（题干 24px、选项 24px）。

## 三、圆角
| 名称 | 大小 | 用途 |
|------|------|------|
| 按钮 | 12px | 所有按钮 |
| 卡片 | 16px | 卡片、弹窗、题目卡 |
| 图片 | 8px | 合影、缩略图 |

> 游戏端容器/大按钮可适当加大到 16-20px，更圆润软萌；编辑端保持 12-16px，利落。

## 六、风格关键词
- **编辑端**：简约、清新、专业 —— 纯白底、信息密度高、克制、左导航选中项淡蓝高亮 + 左竖条。
- **游戏端**：圆润、明亮、绘本风、有童趣 —— 淡黄暖底、超大字、大圆角、一屏一件事。

### MVP 童趣怎么实现（不引插画也能做活）
- **emoji**：🎉🚀📍🏆💡 等（线框图里画的那些），系统自带、零成本、零依赖。
- **简单 CSS 动效**：撒花（CSS animation 或 emoji 飘落）、答对时按钮轻微弹跳、页面切换淡入。
- **禁止**：引粒子库 / 动画库（CLAUDE.md 约束）。撒花用纯 CSS 即可。

### V1 视觉升级（本规范之上叠加，不推翻）
- 自定义插画 / PNG 图标，跟故事线视觉统一。
- 每个 L2 通关庆祝页配独立图片（如太空探险配不同星球）。
- 更丰富的通关庆典动画。
- （以上见 版本路线图.md V1「完整通关庆典」章节）

## 七、给开发的 CSS 变量建议（可直接抄进 :root）

```css
:root {
  /* 主色 */
  --color-primary: #4A90E2;
  --color-primary-dark: #3A7BC8;
  --color-primary-light: #7BB0EE;
  /* 辅助 / 强调 / 错误 */
  --color-accent: #F5A623;
  --color-success: #7ED321;
  --color-error: #FF9F89;
  /* 中性 */
  --color-bg: #FFFFFF;
  --color-bg-card: #F8F9FA;
  --color-text: #2C3E50;
  --color-text-muted: #6C757D;
  --color-border: #E9ECEF;
  /* 游戏端暖背景 */
  --color-bg-game: #FFF9E6;
  /* 圆角 */
  --radius-btn: 12px;
  --radius-card: 16px;
  --radius-img: 8px;
  /* 间距 */
  --space-xs: 8px;
  --space-sm: 16px;
  --space-md: 24px;
  --space-lg: 32px;
  /* 阴影 */
  --shadow-card: 0 2px 8px rgba(0,0,0,0.1);
  --shadow-float: 0 4px 12px rgba(0,0,0,0.15);
}
```

> 编辑端和游戏端各有自己的 CSS 文件，但这套变量两端共用（仅背景色不同：编辑端用 `--color-bg`，游戏端用 `--color-bg-game`）。
