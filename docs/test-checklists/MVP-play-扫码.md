# 功能名：游戏端扫码主流程（M20）

> 覆盖范围：scan.html 扫码界面 + scan-verify.js 验证逻辑

## 前置准备

- 游戏端已导入 JSON（含 ≥1 个探险、≥3 个点位）
- 从 select-level.html → start-level.html → hint.html 正常走到 scan.html
- 准备 3 张打印的二维码：当前站的码、本探险另一站的码、完全无关的码（可随意找个二维码图片）

---

## 桌面 Chrome 测试

### 基本启动流程
- [ ] 打开 scan.html?sessionId=xxx（从 hint.html 点「我找到了！扫码！」进入）
- [ ] 页面显示深色背景 + 「📷 点击开始扫码」大按钮
- [ ] 右上角显示「第 N 站」标签（N = 当前站序号）
- [ ] **摄像头不自动启动**（验证游戏端决策 #4）
- [ ] 「← 返回」可点击 → 跳回 hint.html?sessionId=xxx
- [ ] F12 Console 无红色报错

### 摄像头启动
- [ ] 点「点击开始扫码」→ 浏览器弹出摄像头权限请求
- [ ] 允许后：摄像头画面显示，绿色扫码框出现
- [ ] 启动按钮消失（被摄像头画面取代）
- [ ] 提示文字「把二维码放进框里就行啦～」可见
- [ ] 底部两个灰色按钮始终可见（不被摄像头遮挡）
- [ ] F12 Console 无报错

### 扫到当前站的码（正确）
- [ ] 对准当前站二维码 → 识别后立即停止摄像头
- [ ] 跳转到 challenge.html?sessionId=xxx（M22 前会 404，但能看到 URL 变化，说明判定正确）
- [ ] Console 打印「识别内容：XXXXXX」（6 位数字）

### 扫到本探险其他站的码（错站）
- [ ] 对准另一站的二维码 → 停摄像头
- [ ] 弹出橙黄色卡片：「🤔 这不是你现在要找的点哦！现在要找第 N 站，再看看提示～」
- [ ] 卡片 ~2 秒后自动消失 → 恢复「点击开始扫码」按钮（不自动重启摄像头）

### 扫到完全无关的码（无效码）
- [ ] 对准无关二维码 → 停摄像头
- [ ] 弹出粉红色卡片：「❓ 这不是寻宝游戏的码哦，可能是别的二维码...」
- [ ] 同上，2 秒后回到「点击开始扫码」状态

### 摄像头权限被拒
- [ ] 浏览器弹权限时选「拒绝」→ 显示错误提示（内容含「设置」指引 + 建议用「输入数字码」）
- [ ] 「再试一次」按钮可点击 → 再次触发浏览器权限询问
- [ ] 底部「输入数字码」「找不到二维码?」按钮不报错（点击仅 console.log，M21 实现功能）

### 返回按钮
- [ ] 扫码中途（摄像头运行时）点「← 返回」→ 摄像头停止 → 跳回 hint.html
- [ ] 跳回后 sessionId 参数保持一致（不会进 404）

### 手动构造特殊 URL
- [ ] scan.html（不带 sessionId）→ 自动跳到 select-level.html
- [ ] scan.html?sessionId=不存在的ID → 自动跳到 select-level.html
