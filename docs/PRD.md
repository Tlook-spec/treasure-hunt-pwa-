## 1. 产品概述

### 1.1 一句话定位

**iPad 上离线运行的儿童户外解谜游戏 PWA**——家长在电脑上设计关卡和题库，孩子在 iPad 上扫码答题。

### 1.2 使用场景

家长在电脑上设计关卡、批量录入题库、生成二维码 PDF 并打印；周末在小区、公园等户外场所提前贴好二维码；孩子拿着 iPad 按提示找点位、扫码、回答关于这个位置的趣味题目（数学、英语、自然知识等），完成所有点位后获得徽章和奖励。

### 1.3 核心特性

- **完全离线**：户外无网络也能玩
- **双端体验**（v1.1 新增）：
  - **桌面端**（家长用）：高效设计关卡、批量管理题库、CSV 导入、PDF 打印
  - **iPad 端**（孩子用）：扫码答题、拍照、通关庆典
  - 同一个 PWA，通过 JSON 导入导出同步数据
- **顺序解锁**：必须按家长设计的顺序通关
- **多人协作**：支持单个孩子自己玩，也支持 2-6 个孩子组队游戏
- **自定义内容**：家长可自由创建关卡、添加题目、设计提示
- **正向激励**：多元化奖项、保底奖、通关庆典、合影海报
- **教育融合**：把学科知识融入游戏，复习不枯燥

### 1.4 设计哲学

- **合作 > 竞争**：通关庆典先展示团队总分，再展示个人贡献
- **温和 > 严苛**：答错有容错、求助有机会、孩子不会"卡死"
- **过程 > 结果**：每个孩子至少有一个奖，强调参与感
- **简单 > 复杂**：家长 10 分钟能搭一个关卡，孩子 0 学习成本会玩
- **效率 > 全能**：家长重的事在电脑上做（批量录入、PDF 打印），孩子轻的事在 iPad 上做（玩、扫、答）

## 2. 用户角色

### 2.1 家长（编辑者）

**画像**：4-12 岁孩子的父母，希望陪孩子做有意义的户外活动，对教育有想法。

**典型任务**：
- 在电脑上批量录入题库（用 Excel 整理后 CSV 导入）
- 在电脑上设计关卡、配置点位、绑定题目
- 在电脑上生成二维码 PDF 并直接打印
- 出门前实地贴二维码
- 把数据导出 JSON 文件，传到 iPad 同步
- 游戏过程中陪伴孩子、应急处理

**对 App 的期待**：
- 编辑界面简单清晰，不要太多专业概念
- 电脑大屏幕、键盘录入效率高
- 题库维护直观高效，标签清晰，能根据标签选择进行随机出题
- 题库能复用，并标注某道题是否已经出过（出题的关卡名+已出次数）
- 二维码生成和打印一键搞定
- 数据可备份（不丢失）

### 2.2 孩子（玩家）

**画像**：4-12 岁儿童，认字程度差异大。

**典型任务**：
- 启动游戏，加入队伍
- 看提示找下一个点位
- 扫描二维码
- 答题
- 拍照留念
- 通关后看排行、投票、看海报

**对 App 的期待**：
- 界面友好，看不懂的字能朗读
- 答错不被批评，有鼓励
- 通关有奖励、有仪式感
- 拍的照片好看

- **手动 JSON 导入导出**（电脑端 ↔ iPad 端的同步方式）

**可以延后（V2 及以后）**：
- **GitHub Pages 自动同步**（替代手动 JSON 同步）
- 故事背景文本
- 隐藏奖励点
- 玩家长期档案 + 学科雷达
- 徽章系统
- 关卡分享
- 贴纸滤镜
- 多语言

## 4. 详细功能说明

### 4.1 启动与首屏

**启动页**：
- App 图标 + "Treasure Hunt" 标题 + 简短副标题（如"一起去探险吧！"）
- 一个大按钮："开始游戏"
- 角落有不显眼的"家长入口"（长按 3 秒触发，弹密码框）
- 桌面浏览器打开时检测到大屏幕，可显示提示"检测到桌面端，是否直接进入家长模式？"

**关卡选择页**：
- 显示所有已创建的关卡卡片
- 每张卡片显示：关卡名、点位数、上次游玩时间、缩略图
- 点击卡片进入"开始游戏"流程
- 长按卡片显示"删除"选项

### 4.2 家长模式（编辑端）

#### 4.2.1 进入家长模式

- 启动页长按角落 3 秒
- 弹出密码输入框（密码由家长首次进入时设置）
- 密码错误 3 次锁定 1 分钟
- 进入后界面顶部明显标识"家长模式"

#### 4.2.1.5 家长模式主页（v1.1 新增）

进入家长模式后看到的导航界面：

**桌面端布局**（≥1280px）：
- 左侧导航栏（固定）：
  - 📝 关卡管理
  - 📚 题库管理
  - 📥 数据导入
  - 📤 数据导出
  - ⚙️ 设置（密码、视觉风格等）
  - 🔙 退出家长模式
- 顶部状态栏：「家长模式」标识 + 用户提示
- 中间主区域：默认显示"关卡管理"

**iPad/手机端布局**：
- 主页大按钮卡片（同上述功能）
- 顶部「家长模式」横幅 + 退出按钮
- 友好提示："建议在电脑上进行批量管理"

#### 4.2.2 关卡管理

**关卡列表**：
- 显示所有关卡（卡片或列表形式）
- 操作：新建、编辑、复制、删除、导出、导入

**新建关卡表单**：
- 关卡名称（必填）
- 简介（选填，3-5 行）
- 推荐人数（1-6）
- 推荐年龄段（4-6 / 7-9 / 10-12）
- 主题色（从预设几个选）
- 开场故事（选填，未来 V2 使用）
- 结局故事（选填）

#### 4.2.3 点位编辑

**点位列表**：
- 在关卡详情页显示该关卡的所有点位
- 拖拽排序（决定通关顺序）
- 操作：新建、编辑、复制、删除点位

**点位编辑表单**：
- **基本信息**：
  - 点位序号（自动）
  - 点位名称（仅家长可见，如"小区凉亭"）
  - 家长备忘（仅家长可见，如"贴在凉亭东侧第三根柱子上"）
- **下一点提示**（孩子可见）：
  - 文本框（如"找到 5 号楼旁边的蓝色垃圾桶"）
  - 支持多行
- **题目配置**：
  - 选择题目来源：
    - 从题库中选具体题目（显示已用次数提醒）
    - 从题库标签中随机抽取（如"数学-中等"）
    - 直接为这个点位新建题目（自动入库可勾选）
  - 难度调整（可选）
- **到达后行为**（默认）：
  - 答对题后解锁拍照
  - 拍照后解锁下一点

#### 4.2.4 题库管理

**题库主页**：
- 顶部筛选条：学科、难度、年龄段、使用状态（未使用/已使用 N 次）
- 显示所有题目的表格/卡片
- 每道题显示：题干前 30 字、标签、难度、使用情况
- 使用情况显示（v1.1 新增）：
  - 🔵 「未使用」
  - 🟡 「已用 1 次：周末小区探险」
  - 🟠 「已用 3 次（点击查看引用关卡）」
- 操作：新建、编辑、删除、复制、CSV 导入、CSV 导出、JSON 导出

**题目数据结构**：
```
题目 {
  id: 唯一标识
  学科标签: 数学 | 英语 | 自然 | 地理 | 常识 | 脑筋急转弯 | 其他
  难度: 1星 | 2星 | 3星
  适用年龄: 4-6 | 7-9 | 10-12
  题型: 单选 | 判断 | 多选 | 填空
  题干: 文本 + 可选截图
  选项: [选项A, 选项B, 选项C, 选项D]
  正确答案: 索引
  题目提示: 选填，答错时显示
  解析: 选填，答对后可选展示
  插图: 选填，图片
  使用记录: [{关卡 ID, 关卡名, 引用时间}, ...]
}
```

**MVP 仅支持**：单选、判断
**V1 加入**：多选、填空
**V2 加入**：拖拽连线、图片选择

##### CSV 批量导入流程（v1.1 详细化）

家长在电脑端的核心效率工具。

**步骤 1：下载模板**
- 题库管理页有"📥 下载 CSV 模板"按钮
- 下载得到 `题库模板.csv`（UTF-8 编码）
- 模板列：学科、难度、年龄段、题型、题干、选项A、选项B、选项C、选项D、正确答案、提示、解析、插图文件名

**步骤 2：用 Excel 编辑**
- 家长用 Excel / Google Sheets / WPS 打开模板
- 批量录入题目，每行一道题
- 完成后另存为 CSV（UTF-8 编码，避免中文乱码）

**步骤 3：上传导入**
- 题库管理页 → 「📂 导入 CSV」
- 选择 CSV 文件
- 如有插图，再选择插图所在文件夹（可选）
- 系统预览前 5 行数据让家长确认

**步骤 4：导入结果**
- 显示导入结果："成功 47 道，失败 3 道"
- 失败的会显示原因（如"第 8 行：缺少正确答案"）
- 失败的不入库，家长修正 CSV 后重新导入

#### 4.2.5 二维码生成与打印

**二维码内容编码**：
```
格式：{关卡ID}|{点位ID}|{校验码}
校验码：HMAC(关卡ID + 点位ID, 关卡密钥) 取前 8 位
```

**生成流程**：
- 在点位编辑页有"预览二维码"按钮
- 在关卡详情页有"生成所有二维码"按钮
- 单个二维码可保存图片

**PDF 打印包**：
- 一键导出整关 PDF（A4 大小）
- 每页最多 4 个二维码
- 每个二维码下方标注：
  - 大字：第 X 站
  - 小字：点位名称（如"小区凉亭"）
  - 小字：备忘（如"贴在凉亭东侧"）
- 顶部有关卡名称和日期
- **推荐在电脑上完成 PDF 生成 + 打印**，体验比 iPad 好得多

#### 4.2.6 数据导入导出（v1.1 详细化）

这是 MVP 阶段电脑端 ↔ iPad 端同步的核心机制。

##### 导出

**导出选项**：
- **单关卡导出**：包含该关卡 + 它引用的所有题目（用于分享单个关卡）
- **全部数据导出**：所有关卡 + 全部题库（一份完整备份）
- **仅题库导出**：不含关卡（用于备份题库）
- **CSV 导出**：仅题库，便于 Excel 查看修改

**文件格式**：单个 JSON 文件（不需要 zip），方便邮件/微信/AirDrop 传输。

**导出位置**：
- 桌面浏览器：触发下载到默认下载文件夹
- iPad：可保存到「文件」App、AirDrop 给其他设备

##### 导入

**导入选项**：
- **智能合并**（V1）：检测到已存在的题目/关卡时询问"覆盖/跳过/新建副本"
- **完整覆盖**：先清空再导入（危险操作，二次确认）
- **追加导入**：保留现有数据，添加新内容（MVP 阶段默认行为）

##### 典型工作流（v1.1 强调）

**电脑 → iPad 同步**：
1. 家长在电脑浏览器进入家长模式
2. 完成所有编辑（关卡、题库、二维码）
3. 点击「📤 导出全部数据」→ 得到 JSON 文件
4. 通过 AirDrop / iCloud / 微信文件传输助手把 JSON 传到 iPad
5. iPad 打开 PWA → 进入家长模式 → 「📥 导入数据」→ 选 JSON 文件
6. 同步完成

**iPad → 电脑反向同步**（备份用）：
1. iPad 上偶尔小修小补（如临时调整提示文本）
2. 完成后导出 JSON
3. 传回电脑保存备份

> **V2 升级方向**：自动通过 GitHub Pages 同步，跳过手动文件传输。详见 §9。

#### 4.2.7 应急通关（裁判模式）

- 游戏进行中，长按屏幕右上角 3 秒 + 输密码 → 进入应急菜单
- 选项：
  - 跳过当前点
  - 跳到指定点
  - 强制通关
  - 重置游戏
  - 编辑当前游戏的玩家信息

#### 4.2.8 测试通关模式（v1.1 新增）

家长在关卡详情页有"🧪 测试通关"按钮，用于编辑完关卡后验证：

- 模拟一遍完整游戏流程
- 不需要实地（每个点位都显示二维码图片，可以用另一台手机扫；也可以点"模拟扫到"快速跳过）
- 题目正常显示和答题
- 拍照、归属选择等都可以正常体验
- **不会污染真实数据**（不计入游戏记录、不保存照片）
- 测试结束有"测试报告"：用时、流程是否顺畅、有无异常

用于家长发布关卡前的"自检"，避免出门发现关卡有 bug。

### 4.3 游戏模式（孩子端）

#### 4.3.1 开局流程

**第一步：选关卡**
- 显示所有可玩关卡
- 点击进入

**第二步：人数选择**
- 大字"今天有几个小探险家？"
- 1-6 的数字按钮，点击选择

**第三步：玩家创建（按选定人数依次）**
- 输入名字（限 4 个字）
- 拍头像（前置相机，圆形裁剪）
- 系统自动分配玩家编号（P1-P6）和专属颜色（红橙黄绿蓝紫）

**第四步：队名设置（可选）**
- 输入队名（如"超级侦探队"）
- 可跳过

**第五步：游戏开始**
- 显示队名 + 所有玩家头像
- 大按钮"开始冒险！"

#### 4.3.2 游戏主流程（每个点位）

**1. 提示展示**
- 大字显示当前是第几站（如"第 1 站"）
- 提示文本（如"找到 5 号楼旁边的蓝色垃圾桶"）
- 小喇叭按钮：点击朗读提示
- 放大镜按钮"我找到了！扫码！"

**2. 扫码**
- 全屏摄像头预览
- 中央扫码取景框
- 上方提示"对准二维码"
- 扫到后自动识别

**3. 扫码结果处理**
- **如果是当前应到达的点**：
  - "叮咚"音效 + 撒花动画
  - 进入答题
- **如果是其他点的码**：
  - 友好提示"这不是你现在要找的点哦！再看看提示～"
  - 显示当前提示
  - 不扣分、不算错

**4. 答题**
- 显示题目（含小喇叭朗读按钮、题干截图如果有）
- 显示当前点位编号、剩余求助次数
- 选项按钮（大、易点）

**5. 答题反馈**
- **答对**：
  - "叮咚"音效 + 绿色对勾 + 文字"答对了！"
  - 显示文字"是哪位小探险家答对的呢？"，进入归属选择
- **第 1 次答错**：
  - 温和提示"再想想？"
  - 可重选
- **第 2 次答错**：
  - 温和提示"再想想？" + 显示题目提示（如果有）
  - 可重选
- **第 3 次答错**：
  - 显示正确答案
  - 鼓励文案："没关系，下次记住啦！"
  - 自动算"通过"，0 分

**6. 归属选择**（多人模式）
- "是谁答对的？"
- 显示所有玩家头像
- 点击选中（支持多选）
- 有"大家一起"按钮
- 选完点确认

**7. 拍照留念**
- 解锁相机
- 拍一张
- 照片自动加水印（关卡名、第几站、时间）
- 存到本地（IndexedDB + 系统相册询问）

**8. 下一点提示**
- 显示下一站提示
- 大按钮"前往下一站"

#### 4.3.3 求助机制

- 答题页面右上角"求助"按钮 + 剩余次数（如"求助 3"）
- 点击：
  - 次数 -1
  - 弹出动画"快去问爸爸/妈妈！"
  - 提示朗读题目和选项
  - 不影响计分（但用了求助答对的题，得分会低一些）

#### 4.3.4 连续答对提醒（多人）

- 同一玩家连续 3 题被选为答对者
- 下一题答对时（指定归属前）弹出气泡：
  - "哇！{P1} 已经连续答对 3 题啦，太厉害啦！🌟"
  - "接下来这道题，让其他小伙伴试试吧!"
  - 两个按钮：「让大家试试」「{P1} 继续答」

#### 4.3.5 积分系统

| 情况 | 得分 |
|------|------|
| 一次答对 | 10 分 + "快速思考"标记 |
| 答错 1 次后答对 | 5 分 |
| 用求助答对 | 3 分 |
| 难度 2 星 | ×1.2 |
| 难度 3 星 | ×1.5 |
| 第 3 次答错保底通过 | 0 分 |

### 4.4 通关流程

#### 4.4.1 通关庆典页（多屏）（v1.1 流程调整）

**屏 1：团队总成绩**
- 队名出现 + 队员头像集合出现
- 撒花动画
- 大字："你们一共得了 X 分！"
- 副字："用时 Y 分钟"
- 大按钮"来选出今天的明星们吧 →"

**屏 2：投票环节（详见 §4.4.2）**
- 全员依次为每个奖项投票
- 中途不公布任何结果，保持悬念
- 投票结束后系统自动计算所有奖项

**屏 3：颁奖典礼（v1.1 新流程：所有奖项一起揭晓）**

投票结果计算完成后，进入颁奖典礼。所有奖项**依次**揭晓，动画效果统一，氛围隆重：

- **第一个揭晓**：答题之星 🏆
  - 头像放大居中，金色皇冠从天而降
  - 鼓掌音效
  - 显示答对题数 + 总得分
  - 停顿 2-3 秒让孩子感受荣耀
- **依次揭晓投票奖项**（按家长配置的顺序）：
  - 小飞毛腿奖 🏃 → 获奖者头像放大、银色皇冠
  - 火眼金睛奖 🔍 → 获奖者头像放大、银色皇冠
  - 金点子奖 💡 → 获奖者头像放大、银色皇冠
  - 最佳队友奖 🤝 → 获奖者头像放大、银色皇冠
- **保底奖**（v1.1 自然融入）：
  - 如果有玩家在投票后仍未获得任何奖项，给 ta 颁特别奖：
    - 未来之星 ⭐
    - 最强辅助奖 🤗
  - 同样的隆重揭晓动画，让 ta 也享受荣耀
  - 措辞真诚："虽然今天没拿到投票奖项，但你的坚持就是最棒的！"

**屏 4：全员荣耀合影**
- 所有玩家头像横向排列
- 每个头像上方戴着对应的奖项徽章
- 头像下方显示积分
- 大按钮"看看我们的探险照片 →"

**屏 5：个人积分排行**
- 所有玩家头像横向排列
- 头像下方有积分条（动画从 0 涨到最终分）
- 按分数排序，第二名银皇冠、第三名铜皇冠
- 每个头像下方显示获得的奖项称号
- 大按钮"看看我们的合影 →"

**屏 6：合影墙**
- 显示这次拍的所有照片（九宫格或瀑布流）
- 顶部"探险日记 · {日期}"标题
- 大按钮"导出海报留念 →"

**屏 7：海报导出**
- 自动生成一张精美海报
- 包含：队名、日期、所有玩家头像 + 称号、合影九宫格、总分、用时
- 选项：保存到相册、分享、再来一关、回主页

#### 4.4.2 投票环节（v1.1 明确规则）

**奖项列表**（家长可在编辑关卡时自定义，默认）：
- 🏃 **小飞毛腿奖**：今天跑得最快的人
- 🔍 **火眼金睛奖**：今天最先发现二维码的人
- 💡 **金点子奖**：今天想出最棒办法的人
- 🤝 **最佳队友奖**：今天最爱帮助别人的人

**投票流程**（每个奖项一屏）：

1. 显示奖项名 + 大图标
2. 显示所有候选玩家头像（答题之星头像置灰、标注「已获答题之星」，不可投）
3. 屏幕中央提示「现在请 {P2} 来投票」
4. {P2} 点击想投的头像（**包括自己**）
5. 自动切到下一玩家「现在请 {P3} 来投票」
6. **投票期间不显示前面玩家的选择**，避免互相影响
7. 全员投完后**不立即公布结果**，直接进入下一个奖项

**投票规则（v1.1 采用宽松规则）**：
- 每个奖项每个玩家一票
- 可以投自己
- **同一个孩子可以被多个奖项投中**（不限制每人最多得几个奖）
- 平票则共同获奖

**保底奖触发逻辑（v1.1 明确）**：
- 全部投票结束后，系统检查每个玩家的获奖情况
- 检查依据：答题之星 + 投票奖项
- 如果某个玩家**一个奖都没有**，自动颁特别奖：
  - 未来之星 ⭐
  - 最强辅助奖 🤗
  - （由系统从备选池中随机选一个，或按某种规则分配）
- 保底奖在颁奖典礼中一起揭晓，仪式感与其他奖项相同

### 4.5 暂停与续玩

**自动保存**：
- 游戏中每个关键节点（扫码、答题、归属选择、拍照）自动保存进度

**主动暂停**：
- 游戏中任意时刻点右上角"暂停"
- 弹窗：继续 / 退出
- 选退出 → 状态保留，回到主页

**续玩**：
- 启动 App 时检查：是否有未完成的游戏？
- 有 → 弹窗"上次的游戏还没玩完，要继续吗？"
- 选继续：恢复到上次状态
- 选放弃：清除存档

## 5. 视觉与交互规范

### 5.1 朝向

- **游戏模式**：竖屏优先（iPad / iPhone）
- **家长模式**：桌面横屏优先，平板/手机响应式适配
- 所有页面在主要朝向下体验最优
- 使用响应式 CSS（Flexbox/Grid），不写两套独立布局

### 5.2 颜色

待用户在阶段 1 选定，初始建议：
- 主色：明亮但不刺眼的蓝色或绿色
- 辅助色：与主色互补
- 强调色：黄色或橙色（用于"成功"、"奖励"）
- 错误提示色：用暖色而非刺眼红色

### 5.3 字号

- 标题：28-36px
- 副标题：20-24px
- 正文：16-18px
- 提示：14px
- **最小可点击元素**：60×44px

### 5.4 圆角与阴影

- 按钮：圆角 12-16px
- 卡片：圆角 16-20px
- 图片：圆角 8-12px
- 阴影：柔和、不要太重

### 5.5 动画

- 页面切换：滑动或淡入，300ms
- 按钮点击：轻微缩放反馈
- 通关庆典：撒花、皇冠下落等可以丰富
- 答对反馈：对勾画出动画
- 颁奖：每个奖项揭晓有 2-3 秒"悬念停顿"

### 5.6 文案风格

- 对孩子说话：温暖、鼓励、用"小探险家""小伙伴"等称呼
- 不要批评式语言（如"错了"、"失败"）
- 多用感叹号 + 表情符号
- 短句优于长句

### 5.7 音效

- 扫码成功："叮"
- 答对："叮咚"
- 答错：温和的"哎呀"
- 翻页："咻"
- 通关：鼓点 + 撒花音
- 颁奖：号角声（每个奖项揭晓时）

### 5.8 朗读

- 所有题目和提示文本旁边有小喇叭按钮
- 默认中文，速度稍慢（适合儿童）
- 用浏览器原生 Web Speech API（免费、离线）

### 5.9 响应式断点（v1.1 新增）

| 设备 | 宽度范围 | 主要用途 | 布局特点 |
|------|---------|---------|---------|
| 桌面大屏 | ≥1280px | 家长管理 | 左侧导航 + 主区域，多列布局，大表格 |
| 桌面/平板横屏 | 1024-1279px | 家长管理 | 同上，略紧凑 |
| iPad 横屏 | 768-1023px | 偶尔家长用 | 顶部 Tab 导航 + 单列主区域 |
| iPad 竖屏 | 481-767px | **游戏模式主战场** | 单列、大按钮、纵向流 |
| 手机竖屏 | ≤480px | 备用 | 简化布局 |

**核心原则**：
- 同一份 HTML，通过 CSS 媒体查询切换布局
- 游戏模式在所有设备下都保证可用，但 iPad 竖屏体验最优
- 家长模式在桌面下体验最优，iPad/手机可用但效率较低

## 7. 边界情况与异常处理

### 7.1 编辑模式

- **删除关卡**：弹二次确认，删除后无法恢复（除非有导出文件）
- **删除题目**：检查是否被关卡引用，引用中则不允许删除（提示"该题被 X 个关卡使用"）
- **二维码内容冲突**：每个 Point 的二维码内容必须唯一
- **关卡密码丢失**：提供"重置密钥"功能，但会让旧二维码失效
- **CSV 导入错误**：单行错误不影响其他行；导入完成显示成功/失败统计 + 失败行号和原因
- **CSV 格式不对**：友好提示"请使用标准模板，可从【下载模板】重新下载"

### 7.2 游戏模式

- **扫到别人的二维码**：友好提示，不算错
- **扫到完全无效的二维码**（非本 App 生成的）：提示"这不是寻宝游戏的码哦"
- **答题中切换 App**：恢复后回到答题状态，倒计时（如果有）暂停
- **拍照权限被拒绝**：提示"需要相机权限才能拍照打卡哦~"+ 引导去设置开启
- **设备旋转**：游戏中支持竖屏 ↔ 横屏，但不强制
- **电量过低**：弹温馨提示"iPad 没电啦，建议先充电"
- **存储空间不足**：拍照前检查，提示家长清理

### 7.3 多人模式

- **中途有人退出**：保留积分，结算时标"提前退出"
- **中途加人**：家长模式应急菜单中添加
- **所有玩家投票都投自己**：平票共享获奖
- **没人投某个奖项**：奖项空缺，不强制颁出
- **某玩家所有奖项都没获得**：自动获得保底奖

### 7.4 数据安全

- **iOS 清理 PWA 数据风险**：
  - 提示用户至少每周打开一次
  - 每次大改后引导用户导出备份
  - 拍的照片同步存到系统相册（双保险）
  - **强烈建议**：所有重要数据先在电脑端编辑并保留 JSON 备份，iPad 只是消费端
- **导出文件丢失**：可考虑同步到 iCloud Drive / OneDrive / 坚果云（家长手动）

### 7.5 双端同步（v1.1 新增）

- **导入 JSON 版本不兼容**：检查 JSON 文件的版本号，不兼容时提示"该文件由更新版本的 App 创建，请先升级 App"
- **导入数据冲突**：智能合并模式询问用户处理方式
- **iPad 上编辑后忘记导出回电脑**：可考虑加"上次导出时间"提醒，每次进入家长模式时检查

## 8. 性能要求

- **首屏加载**：5 秒内（离线状态）

- **题库容量**：至少支持 1000 道题
- **关卡数量**：至少支持 50 个关卡
