# 手动测试 Checklist：L1 探险 CRUD（M07）

## 桌面浏览器测试（Windows + Chrome）

### 列表页基础
- [ ] 访问 `http://localhost:8000/admin/#/levels`，显示「探险管理」标题 + 「➕ 新建探险」按钮
- [ ] 初始无数据时显示空状态（🗺️ + 文字提示）
- [ ] F12 Console 无红色错误

### 新建探险
- [ ] 点击「新建探险」→ 弹出弹窗，标题显示「新建探险」
- [ ] 5 个字段全部可以填写/选择：名称、简介、推荐人数、推荐年龄段、主题色
- [ ] 主题色 6 个色块可点击，点中的有边框高亮
- [ ] 名称留空点「保存」→ 弹出 alert 提示，不保存
- [ ] 填入名称后点「保存」→ 弹窗关闭，列表出现新卡片
- [ ] 卡片顶部彩条颜色与选中的主题色一致
- [ ] 卡片显示：名称、简介（无则「无简介」）、点位数 0、人数、年龄段、日期
- [ ] 点「取消」或弹窗遮罩 → 不保存，弹窗关闭

### 编辑探险
- [ ] 点卡片上「✏️ 编辑」→ 弹窗打开，5 个字段已填入当前值
- [ ] 修改名称后保存 → 卡片更新，「最近编辑」时间更新
- [ ] 保存后 createdAt 不变（重新进编辑弹窗，创建时间与第一次一致）

### 删除探险
- [ ] 点卡片上「🗑️ 删除」→ 弹出确认弹窗，包含探险名称
- [ ] 点「取消」→ 不删除，弹窗关闭
- [ ] 点「确认删除」→ 探险从列表消失
- [ ] 列表只剩一个时删除 → 变为空状态

### 复制探险
- [ ] 点「📋 复制」→ 列表顶部出现名为「原名（副本）」的新卡片
- [ ] 副本的 id 与原始不同（F12 IndexedDB 可查验）
- [ ] 副本的 createdAt、updatedAt 更新为复制时的时间

### 详情页
- [ ] 点「👁️ 详情」→ 切换到详情视图，隐藏列表
- [ ] 详情显示：标题、简介、点位数、推荐人数、年龄、主题色色块、创建/编辑时间
- [ ] 「← 返回列表」按钮 → 返回卡片列表
- [ ] 详情页顶部「✏️ 编辑」→ 弹出编辑弹窗，保存后详情内容刷新
- [ ] 详情页顶部「📋 复制」→ 复制后返回列表，出现副本卡片
- [ ] 详情页顶部「🗑️ 删除」→ 确认后返回列表，探险消失
- [ ] 「➕ 新建点位」按钮处于 disabled 状态（M08 实现）
- [ ] 「打印工具」两个按钮均为 disabled 状态（M11/M12 实现）

### 多条数据
- [ ] 连续新建 3 个探险 → 3 张卡片正常显示
- [ ] 刷新页面 → 数据持久化，3 张卡片仍在

### Console 检查
- [ ] 全程（建→编→删→复制→查看详情→返回）无红色错误

## 移动端测试（Chrome DevTools 模拟 375px 宽度）

- [ ] 卡片以单列显示，文字不溢出
- [ ] 弹窗宽度不超出屏幕
- [ ] 表单两列（推荐人数/年龄段）不溢出
- [ ] 详情页按钮行自动换行，不挤压

## 注意事项

- **暂不需要 iPad 测试**：编辑端不做 PWA，iPad Safari 浏览器访问即可，M15 联调时统一测
- 若 IndexedDB 需要清空，DevTools → Application → IndexedDB → `treasure-hunt-admin-db` → 右键 Delete
