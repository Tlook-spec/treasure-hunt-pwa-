# 功能名：M11 二维码生成 + 打印

## 前置条件

- 编辑端已有至少 1 个探险（L1），其中至少有 2 个已保存的点位（L2）
- 点位都有名称和 6 位数字码

---

## 单张 QR 预览 + 下载（point-form.js）

### 桌面 Chrome 测试

- [ ] 进入某点位编辑页（编辑已存在的点位）→「🔲 预览二维码」按钮**可点**
- [ ] 新建点位（未保存）时 →「预览二维码」按钮**置灰禁用**
- [ ] 点击「预览二维码」→ 弹窗出现，显示：点位名、QR 图、6 位短码
- [ ] QR 图黑白清晰、比例正常（不变形）
- [ ] 手机扫描弹窗里的 QR 图 → 扫出 6 位数字码（与显示一致）
- [ ] 点击「💾 下载图片」→ 浏览器下载 `.gif` 文件
- [ ] 下载文件命名格式正确：`探险名_第N站_点位名_码xxxxxx.gif`
- [ ] 用手机扫下载的 GIF 文件 → 扫出正确 6 位码
- [ ] 点「✕」或「关闭」→ 弹窗关闭
- [ ] 点弹窗遮罩区域 → 弹窗关闭
- [ ] F12 Console 无红色错误

### iPad Safari 测试

- [ ] 同上各步骤均正常
- [ ] 下载图片 → iPad 相册或文件 App 可查看

## 全部 QR 打印预览页（print-qr.html）

### 桌面 Chrome 测试 — 页面加载

- [ ] 在 L1 详情页点「🖨️ 打印二维码」→ **新标签页**打开打印预览
- [ ] 页面顶部显示 iPad 打印警告横幅
- [ ] 页面标题正确（探险名 · 打印二维码）
- [ ] 所有点位的 QR 卡片都渲染出来，数量与点位数一致
- [ ] 每张卡片包含：QR 码（SVG）、第 X 站、点位名、6 位短码
- [ ] 有「家长备忘」的点位显示备忘文字，无备忘的不显示
- [ ] QR 码清晰可辨（SVG 向量，无锯齿）
- [ ] F12 Console 无红色错误

### 桌面 Chrome 测试 — 打印

- [ ] 点「🖨️ 立即打印」→ 打印对话框出现
- [ ] 打印预览中：工具栏和 iPad 警告**不出现**
- [ ] 打印预览中：QR 卡片 2 列布局
- [ ] 第 5 张及之后自动换页（4 张一页）
- [ ] QR 码打印出来黑白清晰，实际打印后用手机扫码 → 扫出正确码
- [ ] 「← 返回」按钮 → 回到上一页

### 桌面 Chrome「另存为 PDF」

- [ ] 打印对话框选「另存为 PDF」→ 生成 PDF
- [ ] PDF 排版：每页 4 张、2 列
- [ ] PDF 中 QR 码清晰（SVG 向量放大不糊）
- [ ] PDF 中无工具栏和警告横幅

### iPad Safari 测试

- [ ] iPad 打开打印预览页，警告横幅正常显示
- [ ] QR 卡片正常渲染
- [ ] 点「立即打印」→ iPad 打印对话框出现
- [ ] 选「另存为 PDF」→ PDF 排版可接受（允许轻微差异）
- [ ] PDF 中 QR 码可扫出

---

## 异常情况

- [ ] 直接访问 `print-qr.html`（不带 levelId）→ 页面显示友好错误提示
- [ ] 传入不存在的 levelId → 页面显示友好错误提示
- [ ] 探险下无点位 → 页面显示「还没有点位」提示
